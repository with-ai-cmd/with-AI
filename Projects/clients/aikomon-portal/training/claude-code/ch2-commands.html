基本コマンド

ターミナル操作、スラッシュコマンド、権限管理、トラブル復旧まで。

ターミナル基礎

ターミナルとは、コンピュータと文字で会話するための画面です。ボタンをクリックする代わりに、文字を入力して操作します。

画面には「プロンプト」と呼ばれる記号が表示されます。これは「入力してください」という合図です。

環境プロンプト記号
Mac / Linux (zsh / bash)$ または %
Windows (cmd)>
Windows (PowerShell)PS>
プロンプト記号は入力する必要はありません。コマンドだけを入力してください。

基本的な流れは次の通りです。

1

プロンプトが表示される

2

コマンドを入力する

3

Enter を押す

4

結果が表示される

cd -- 移動する

cd は「change directory」の略で、フォルダを移動するコマンドです。ファイルを操作する前に、目的のフォルダに移動する必要があります。

デスクトップに移動

cd Desktop

現在いる場所の中にある「Desktop」フォルダに移動します。

1つ上のフォルダに戻る

cd ..

.. は「1つ上の階層」を意味します。間違ったフォルダに入ったときに使います。

ホームフォルダに戻る

cd ~

~（チルダ）は「ホームフォルダ」を意味します。どこにいても一発で戻れます。

cd は最もよく使うコマンドです。迷ったらまず cd ~ でホームに戻りましょう。

mkdir -- フォルダ作成

mkdir は「make directory」の略で、新しいフォルダを作成するコマンドです。

mkdir my-project

現在いるフォルダの中に「my-project」という名前のフォルダが作られます。

フォルダ名にスペースを入れないのがコツです。代わりにハイフン - を使いましょう。

作成して移動する（よくある操作）

# フォルダを作って、その中に移動
mkdir my-project
cd my-project

ls / dir -- 一覧表示

今いるフォルダの中身を一覧表示するコマンドです。OS によってコマンドが異なります。

Mac / Linux

ls

ファイルとフォルダの名前が表示されます。

Windows（cmd / PowerShell）

dir

詳細表示（Mac）

ls -la

隠しファイル（. で始まるファイル）を含め、サイズや更新日時も表示します。CLAUDE.md や .claude フォルダの確認に便利です。

Mac: ls でファイル一覧
Windows: dir でファイル一覧

その他のコマンド

今いる場所を確認する

Mac / Linux

pwd（print working directory）

Windows

cd（引数なしで現在地を表示）

# Mac
pwd
# Windows（引数なし）
cd

「自分が今どこにいるか分からない」と思ったらすぐに実行しましょう。

画面をきれいにする

Mac / Linux

clear

Windows

cls

画面がいっぱいになったら実行すると、見やすくなります。

キーボード操作

ターミナルや Claude Code を使うときによく使うキー操作です。

Enter

コマンドを実行する / メッセージを送信する

Esc

Claude Code で複数行入力モードを切り替える

↑

前に実行したコマンドを呼び出す（履歴）

Ctrl + C

実行中の処理を中断する（困ったらこれ）

Cmd / Ctrl + `

Cursor でターミナルを開く / 閉じる

Tab

ファイル名やフォルダ名を自動補完する

Tab キーは最も便利なキーです。フォルダ名の最初の数文字を入力して Tab を押すと、残りを自動で補完してくれます。

スラッシュコマンド

Claude Code の中で使える特別なコマンドです。通常のターミナルコマンドとは異なり、Claude Code が起動している状態で使います。

/help

/help

使い方のヘルプを表示します。困ったらまずこれを実行してください。

/compact

/compact

会話の履歴を圧縮します。長い会話をしていて動作が遅くなったときに有効です。

/status

/status

現在のログイン状態やアカウント情報を確認できます。

/login

/login

認証が切れたときに再ログインできます。ブラウザが開きます。

/clear

/clear

会話をリセットして最初からやり直します。Claude の返答がおかしくなったときに便利です。

スラッシュコマンドは Claude Code が起動中（プロンプトが > のとき）にのみ使えます。通常のターミナルでは動きません。

権限管理

Claude Code がファイルを編集したりコマンドを実行するとき、許可を求めてきます。3つの選択肢があります。

Allow once

今回だけ許可する。毎回確認が出ます。慎重に進めたいときに。

Always allow

今後同じ操作は自動で許可する。信頼できる操作に。

Deny

拒否する。意図しない操作を止めたいときに。

/permissions

現在の権限設定を確認・変更できます。

/permissions

過去に許可した内容の一覧を確認できます。

よく分からない操作は「Allow once」を選びましょう。慣れてきたら「Always allow」に切り替えると効率的です。

再起動・復旧

Claude Code が固まったり、おかしな動作をしたときの対処法です。軽い順に試してください。

1

Ctrl + C で中断 → claude で再起動

処理が止まらないとき。最も軽い対処法です。

2

/clear で会話をリセット

返答がおかしい、話が噛み合わないとき。会話履歴がクリアされます。

3

claude doctor で自動診断

原因が分からないとき。問題を自動で検出して教えてくれます。

4

Cursor を完全に再起動

上記で解決しないとき。Cursor を閉じて開き直してください。

# レベル1: 中断して再起動
# Ctrl+C を押してから:
claude

# レベル3: 自動診断
claude doctor
ほとんどの問題はレベル1（Ctrl+C → claude）で解決します。
