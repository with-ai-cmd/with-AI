実務活用テクニック

プロンプトの書き方からファイル操作、Git連携まで。実務で使えるスキルを身につけよう。

プロンプトの書き方

Claude への指示（プロンプト）は、書き方次第で回答の質が大きく変わります。

5W1H を意識する

以下の情報を含めると、Claude は的確に回答できます。

要素質問例
Who（誰が）誰の立場で？営業担当として
What（何を）何を作る？提案メールを作成
Why（なぜ）目的は？新規顧客を獲得するため
When（いつ）期限・タイミングは？明日の午前中に送信
Where（どこで）どの場面で使う？初回アプローチ
How（どうやって）形式・条件は？300文字以内で丁寧に

万能テンプレート

迷ったら、このテンプレートに当てはめてみてください。

○○として、△△のために、□□を作って。

条件:
- 条件1
- 条件2
- 条件3

具体例:

営業マネージャーとして、チームの成績を上司に報告するために、
先月の営業レポートを作って。

条件:
- A4で1枚に収まる分量
- グラフは不要、数字の箇条書きで
- 改善点と来月の施策も含める

ステップごとに指示する

複雑な作業は、一度に全部頼まず段階的に進めるのがコツです。

1

まず全体の構成を作ってもらう

2

構成を確認してOKなら、各セクションを書いてもらう

3

最後に全体を見直して修正を依頼する

一度に1000文字の指示を書くより、3回に分けて対話するほうが良い結果になります。

良い例 vs 悪い例

同じ目的でも、書き方で結果が大きく変わります。5つの比較で違いを確認しましょう。

比較 1: メール作成

悪い例

メール書いて

良い例

株式会社ABCの田中様に、来週の打ち合わせ日程調整のメールを書いて。候補は火曜と木曜の午後。丁寧なビジネス文体で。

比較 2: 資料作成

悪い例

いい感じの資料作って

良い例

社内向けAI導入提案書を作って。対象は経営層。構成は「現状の課題 → AI導入のメリット → 導入計画 → 予算」の4部構成で、各セクション3行程度。

比較 3: 翻訳

悪い例

これ英語にして

良い例

以下の文章をビジネス英語に翻訳して。取引先への正式な文書なので、フォーマルな表現で。原文と訳文を並べて表示して。

比較 4: 要約

悪い例

まとめて

良い例

この議事録を「決定事項」「TODO（担当者と期限つき）」「次回予定」の3つに分けて箇条書きで要約して。

比較 5: 修正依頼

悪い例

もっと良くして

良い例

この文章の冒頭を、読み手の興味を引くような質問形式に変えて。全体のトーンはもう少しカジュアルに。

共通するポイント: 具体的に、条件を明記、形式を指定。この3つを意識するだけで回答品質が上がります。

ファイルを作る

Claude はファイルの作成もできます。テキストファイル、Markdownファイルなど、自然な言葉で作れます。

1つのファイルを作る

meeting-notes.md を作成して。
内容は今日の定例会議のテンプレートで。
Claude がファイルを作成すると、ターミナルにファイル名とパスが表示されます。

複数のファイルを一度に作る

以下の3つのファイルを作成して。

1. README.md — プロジェクトの概要説明
2. TODO.md — タスクリスト（空のテンプレート）
3. CHANGELOG.md — 変更履歴（初回エントリだけ）

既存の内容をファイルにする

さっき作ってくれた営業メールの内容を
mail-draft.md として保存して
ファイル名に日本語は使えますが、英語・小文字・ハイフン区切りのほうがトラブルが少ないです。
Claude がファイルを作成・編集するときは確認メッセージが出ます。内容を確認してから許可してください。

ファイルを読む・編集

既存のファイルを Claude に読み込ませたり、編集してもらうことができます。

ファイルを読む

README.md の内容を説明して
このフォルダにあるファイルの一覧を見せて

ファイルを編集する

README.md の3行目を「最終更新: 2026年4月」に修正して
TODO.md の完了したタスクに取り消し線をつけて

ファイルの内容を活用する

meeting-notes.md を読んで、決定事項だけ抜き出して
report.md の内容を英語に翻訳して、report-en.md として保存して
Claude はフォルダ内のファイルを自動的に認識します。ファイル名を伝えるだけで、読み込みから編集まで行えます。

Git 連携

Git は「ファイルのセーブポイント」を作る仕組みです。ゲームのセーブと同じように、いつでも前の状態に戻れます。

Git の基本イメージ

セーブ 1

最初のバージョン

セーブ 2

文章を追加した

セーブ 3

表現を修正した

セーブ 4

最終版

Git を使えば「報告書_v1」「報告書_v2_最終」「報告書_v3_本当の最終」のようなファイル名の地獄から解放されます。

初期設定（最初の1回だけ）

git init
これでフォルダが Git 管理下になります。Claude がファイルの変更履歴を追えるようになります。

Claude が自動でやってくれること

Claude Code は、ファイルを変更するたびに自動でセーブポイント（コミット）を作ります。

# Claude にファイル作成を頼むと...
README.md を作成して

# Claude は裏側で自動的に以下を実行する:
# git add README.md
# git commit -m "Create README.md"
Git のコマンドを覚える必要はありません。Claude が裏側で自動的に管理してくれます。

前の状態に戻す

「さっきの変更を取り消したい」というときも Claude に頼めます。

さっきの変更を元に戻して
README.md を2つ前のバージョンに戻して
Git で戻せるのは、セーブポイント（コミット）が作られた変更だけです。こまめにコミットする習慣をつけましょう。

よくある失敗

よくあるつまずきポイントと、その解決策を5つ紹介します。

失敗 1: 指示が曖昧すぎる

問題

「いい感じにして」「もっと良くして」と頼んで、期待と違う結果になる。

解決策

何がどう良くないのか具体的に伝える。「冒頭の挨拶をもう少しカジュアルに」「箇条書きを3つに減らして」のように。

失敗 2: 一度に大量の指示を出す

問題

10個の条件を一度に伝えて、一部が無視される。

解決策

3〜5個ずつ段階的に指示する。まず骨格を作り、次に詳細を追加する流れが効果的。

失敗 3: 前の会話を忘れている

問題

長い会話の後半で、最初に伝えた条件が反映されなくなる。

解決策

/compact で会話を整理する。または重要な条件は再度伝える。CLAUDE.md に書いておけば常に適用される。

失敗 4: 間違ったフォルダで作業

問題

別のフォルダで Claude を起動してしまい、関係ないファイルが変更される。

解決策

Claude 起動前に pwd で現在地を確認する習慣をつける。エージェントを使う場合は必ず対象フォルダに移動してから。

失敗 5: 確認せずに許可する

問題

Claude のファイル変更を確認せずに全て許可して、意図しない変更が入る。

解決策

変更内容が表示されたら、必ず目を通してから許可する。不安なときは拒否して、指示を修正する。

効率化 Tips

日々の作業をもっと速く、もっと楽にするためのテクニック集です。

1. CLAUDE.md を活用する

繰り返し伝えている指示は CLAUDE.md に書きましょう。毎回伝える手間がなくなります。

# CLAUDE.md に書いておくと便利な内容

## 基本ルール
- 日本語で回答する
- 箇条書きを多用する
- 専門用語は使わず平易な言葉で説明する
- 出力は必ず Markdown 形式にする
「3回以上同じ指示をしたら CLAUDE.md に書く」をルールにしましょう。

2. 良いプロンプトを保存する

うまくいったプロンプトは、テキストファイルに保存しておきましょう。

# prompts.md に保存する例

## 議事録テンプレ
「以下の会議メモを、決定事項・TODO・次回予定の3つに分けて
箇条書きで整理して。TODOには担当者と期限をつけて。」

## メール作成テンプレ
「○○社の△△様に、□□の件でビジネスメールを作成して。
丁寧な敬語で、本文は3段落構成、件名は20文字以内。」

## 要約テンプレ
「この文章を3行で要約して。重要なポイントは太字にして。」

3. /compact を定期的に使う

会話が長くなると Claude の応答速度が落ちたり、前の指示を忘れやすくなります。

# 会話を圧縮して軽くする
/compact
目安として、20〜30回やりとりしたら /compact を実行しましょう。タスクの切り替え時も効果的です。

4. フォルダを整理する

プロジェクトごと、エージェントごとにフォルダを分けると、作業がスムーズになります。

Desktop/
  work/
    project-a/         # プロジェクトA
      CLAUDE.md
    project-b/         # プロジェクトB
      CLAUDE.md
  agents/
    mail-agent/        # メール作成
      CLAUDE.md
    report-agent/      # レポート作成
      CLAUDE.md

5. 作業の型を作る

毎日の作業を「型」にして、Claude に効率よく頼みましょう。

1

朝: 「今日のTODOを整理して」（タスク整理エージェント）

2

日中: 必要に応じてエージェントを切り替えて作業

3

夕方: 「今日の作業内容を日報にまとめて」（日報エージェント）

最初は1つのエージェントから始めて、慣れたら徐々に増やしていくのがおすすめです。

まとめチェックリスト

  • CLAUDE.md に繰り返しの指示を書いておく
  • 良いプロンプトはファイルに保存する
  • /compact で会話を定期的に整理する
  • フォルダを目的別に整理する
  • 毎日の作業を「型」にして効率化する
