API連携

外部サービスと Claude Code をつなげて、業務を自動化する方法を学びます。

API とは

API（Application Programming Interface）は、ソフトウェア同士が会話するための「窓口」です。

レストランで例えると

API の仕組みは、レストランの注文に似ています。

1

あなた（アプリ）がメニューを見て注文する

2

ウェイター（API）が注文をキッチンに伝える

3

キッチン（外部サービス）が料理を作って返す

あなたはキッチンの中に入る必要はありません。ウェイター（API）に頼めば、正しい形で結果が届きます。

なぜ API が重要なのか

自動化

手作業を減らし、繰り返し作業をプログラムに任せられる

連携

異なるサービス同士をつなげて、データをやり取りできる

拡張

既存のサービスに新しい機能を追加できる

効率化

人が画面を操作しなくても、裏側で処理が進む

Anthropic API

Anthropic API は、Claude の能力をプログラムから利用するための窓口です。

何ができるのか

テキスト生成

プログラムから Claude に質問して回答を得る

アプリ組み込み

自社の Web アプリやツールに AI 機能を追加する

バッチ処理

大量のデータを一括で AI に処理させる

カスタム連携

社内システムと Claude を直接つなげる

API コンソール

Anthropic API の管理画面は以下の URL からアクセスできます。

https://console.anthropic.com
API コンソールでは、API キーの発行、使用量の確認、請求管理などが行えます。

API キーの管理

API キーは、あなたが正規のユーザーであることを証明する「鍵」です。

API キーの取得手順

1

console.anthropic.com にログイン

2

左メニューの API Keys をクリック

3

Create Key をクリック → 名前を付ける

4

表示されたキーを すぐにコピーして安全な場所に保存

一度閉じると二度と表示されません

絶対にやってはいけないこと: API キーをソースコードに直接書く、チャットで共有する、Git にコミットする

安全な保管方法: .env ファイル

API キーは .env ファイルに保存し、コードからは環境変数として読み込みます。

# .env ファイルの例
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxx
.env ファイルは必ず .gitignore に追加してください。Git にアップロードされると、キーが漏洩します。

MCP サーバーとは

MCP（Model Context Protocol）は、Claude Code に「手」を与える仕組みです。

MCP の役割

通常の Claude Code はテキストの読み書きが中心ですが、MCP サーバーを接続すると、外部サービスを直接操作できるようになります。

MCP なし

Claude Code → テキスト出力のみ

MCP あり

Claude Code → Notion に書き込み、メール送信、カレンダー登録...

仕組みのイメージ

Claude Code（頭脳）
    |
    v
MCP サーバー（手）
    |
    +---> Notion（データベース操作）
    +---> Gmail（メール送受信）
    +---> Slack（メッセージ送信）
    +---> Calendar（予定管理）

.mcp.json ファイル

MCP サーバーの設定は、プロジェクトルートの .mcp.json ファイルに記述します。

// .mcp.json の基本構造
{
  "mcpServers": {
    "サーバー名": {
      "command": "実行コマンド",
      "args": ["引数1", "引数2"],
      "env": {
        "API_KEY": "キー値"
      }
    }
  }
}
MCP サーバーは必要なものだけを追加してください。不要なサーバーを接続すると、意図しない操作のリスクが高まります。

Notion 連携

Notion MCP サーバーを使うと、Claude Code から直接 Notion のページやデータベースを操作できます。

できること

ページ作成・編集

議事録や日報を自動で作成

データベース検索

タスクやプロジェクトを検索・集計

コメント追加

ページにコメントを自動で追加

データ更新

ステータスや担当者を一括変更

設定例

// .mcp.json -- Notion 設定例
{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["-y", "@notionhq/notion-mcp-server"],
      "env": {
        "OPENAPI_MCP_HEADERS": "{\"Authorization\": \"Bearer ntn_xxxxx\", \"Notion-Version\": \"2022-06-28\"}"
      }
    }
  }
}
ntn_xxxxx の部分を、Notion のインテグレーションで取得した API キーに置き換えてください。

Gmail 連携

Gmail MCP サーバーを使うと、Claude Code からメールの読み取りや下書き作成ができます。

できること

メール検索

条件を指定してメールを検索

メール読み取り

指定したメールの内容を取得

下書き作成

返信や新規メールの下書きを作成

ラベル管理

ラベル一覧の取得

設定例

// .mcp.json -- Gmail 設定例
{
  "mcpServers": {
    "gmail": {
      "command": "npx",
      "args": ["-y", "@anthropic/gmail-mcp-server"],
      "env": {
        "GMAIL_CLIENT_ID": "your-client-id",
        "GMAIL_CLIENT_SECRET": "your-client-secret",
        "GMAIL_REFRESH_TOKEN": "your-refresh-token"
      }
    }
  }
}
Gmail 連携では Google Cloud Console での OAuth 設定が必要です。管理者に相談してください。

Slack 連携

Slack MCP サーバーを使うと、Claude Code からチャンネルへのメッセージ送信や情報取得ができます。

できること

メッセージ送信

指定チャンネルにメッセージを投稿

メッセージ検索

キーワードで過去の会話を検索

チャンネル一覧

参加可能なチャンネルを取得

スレッド返信

既存のスレッドに返信を追加

設定例

// .mcp.json -- Slack 設定例
{
  "mcpServers": {
    "slack": {
      "command": "npx",
      "args": ["-y", "@anthropic/slack-mcp-server"],
      "env": {
        "SLACK_BOT_TOKEN": "xoxb-xxxxxxxxxxxx",
        "SLACK_TEAM_ID": "T0XXXXXXXXX"
      }
    }
  }
}
Slack Bot Token は Slack App の管理画面（api.slack.com/apps）で取得できます。

Calendar 連携

Google Calendar MCP サーバーを使うと、Claude Code から予定の確認や作成ができます。

できること

予定の確認

今日・今週の予定を一覧表示

予定の作成

新しいミーティングを登録

空き時間検索

参加者全員の空き時間を検索

予定の更新

既存の予定の時間や場所を変更

設定例

// .mcp.json -- Calendar 設定例
{
  "mcpServers": {
    "google-calendar": {
      "command": "npx",
      "args": ["-y", "@anthropic/google-calendar-mcp-server"],
      "env": {
        "GCAL_CLIENT_ID": "your-client-id",
        "GCAL_CLIENT_SECRET": "your-client-secret",
        "GCAL_REFRESH_TOKEN": "your-refresh-token"
      }
    }
  }
}
Calendar 連携も Gmail と同様に Google Cloud Console での OAuth 設定が必要です。

セキュリティ

API 連携を安全に運用するためのルールをまとめます。

API キー管理の鉄則

ルール理由
コードに直接書かないGit に残り、漏洩リスクが高い
.env ファイルで管理する環境変数として安全に読み込める
.gitignore に .env を追加Git にアップロードされることを防ぐ
キーを定期的にローテーション万が一漏洩しても被害を限定できる
チャットやメールで共有しない第三者に傍受されるリスクがある

.gitignore の設定

# .gitignore に追加すべき項目
.env
.env.local
.env.production
*.key
*.pem

最小権限の原則

API キーやサービスアカウントには、必要最低限の権限だけを付与してください。

良い例

Notion: 特定のデータベースのみ読み書き可能

悪い例

Notion: ワークスペース全体の管理者権限

セキュリティチェックリスト

  • API キーは .env ファイルで管理している
  • .env は .gitignore に追加済み
  • 各サービスの権限は最小限に設定している
  • 不要な MCP サーバーは接続していない
  • API キーを定期的にローテーションしている
  • チームメンバーにセキュリティルールを共有している
