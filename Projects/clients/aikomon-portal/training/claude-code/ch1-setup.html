Claude Code 導入ガイド

Mac / Windows 全パターン対応。インストールからログインまで。

あなたの環境に合ったパターンを選んでください。

パターンCursorGitNode.js

事前チェック Mac

macOS バージョン確認（13.0以上が必要）

sw_vers
成功: ProductVersion が 13.0 以上
失敗: macOS をアップデートしてください

ネットワーク接続確認

curl -sI https://claude.ai | head -1
HTTP/2 200 → 接続 OK
エラー → Wi-Fi / VPN OFF / テザリング

Cursor インストール Mac

1

cursor.com をブラウザで開く

2

Download をクリック（Mac が自動選択）

3

ダウンロードされた .dmg ファイルを開く

4

Cursor を Applications フォルダにドラッグ

「開発元が未確認」と出た場合 → システム設定 → セキュリティ → 「このまま開く」

ターミナルを開く Mac

Cursor の中にターミナル（黒い画面）を表示します。

方法A: メニュー

上部 → Terminal → New Terminal

方法B: ショートカット

Cmd + `

` キーはキーボード左上、数字 1 の左隣です。

Claude Code インストール Mac

以下をコピーしてターミナルに貼り付け（Cmd+V）→ Enter

curl -fsSL https://claude.ai/install.sh | bash
✓ Claude Code installed successfully! と表示されれば完了
数秒〜1分ほどかかります。途中で止まっても待ってください。

インストール確認 Mac

claude --version
Claude Code v1.x.x と表示されれば成功

ヘルスチェック（任意）

claude doctor
command not found → ターミナルを閉じて開き直す → 再実行

ログイン（認証）Mac

1

claude と入力 → Enter

2

「Use Claude.ai」を選択 → Enter

矢印キー ↑↓ で選択

3

ブラウザが自動で開く → Claude アカウントでログイン

4

「許可」をクリック

5

ターミナルで /status → アカウント情報が出れば完了

ブラウザが開かない → ターミナルに表示された URL を手動でブラウザに貼り付け

Xcode Command Line Tools

初めて git init を実行するとポップアップが出ることがあります。

「インストール」をクリック → 数分待つ → 完了後に再度 git init。初回のみです。

Cursor を使わず、Mac 標準のターミナルで使う場合。

Spotlight

Cmd+Space →「ターミナル」→ Enter

Finder

アプリケーション → ユーティリティ → ターミナル

ターミナルを開いた後の手順は Mac × Cursor と全く同じです。
# インストール
curl -fsSL https://claude.ai/install.sh | bash

# 確認
claude --version

# 認証

事前チェック Windows

Git for Windows の確認（必須）

where.exe git
パスが表示 → Git OK
何も出ない → Git 未インストール
Git がない場合: git-scm.com → Download → 全てデフォルト → Cursor 再起動

実行ポリシーの確認

Get-ExecutionPolicy
RemoteSigned / Unrestricted / Bypass → OK
Restricted → 下のコマンドを実行
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

→「Y」を入力 → Enter

ネットワーク接続確認

Test-Connection claude.ai -Count 1
Status: Success → 接続 OK

ターミナル確認 Windows

Cursor のターミナルが PowerShell であることを確認。

上部に「powershell」→ OK
「cmd」→ 切り替え必要
切り替え: ターミナル右上「＋」横の「▼」→「PowerShell」を選択

Claude Code インストール Windows

PowerShell の場合

irm https://claude.ai/install.ps1 | iex
✓ Claude Code installed successfully! → 成功
「irm が認識されません」→ cmd を使っている → PowerShell に切り替え

cmd（コマンドプロンプト）の場合

curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

確認 & 認証 Windows

Mac と同じ手順です。

# バージョン確認
claude --version

# ヘルスチェック
claude doctor

# 認証（ブラウザが開く）
claude

# ログイン状態確認（Claude Code 内で）
/status
'claude' は認識されていません → ターミナルを閉じて開き直す → Cursor 再起動

PowerShell 単体 / WSL Windows

PowerShell 単体

スタートメニュー →「PowerShell」と入力 → クリック。以降の手順は Cursor (PS) と同じ。

WSL（Windows Subsystem for Linux）

# PowerShell を管理者として実行:
wsl --install

→ PC 再起動 → Ubuntu 起動 → ユーザー名 & パスワード設定

# WSL 内で（Mac と同じ）
curl -fsSL https://claude.ai/install.sh | bash
claude --version
claude

保存場所 & PATH 共通

インストール先

OS場所
Mac / Linux / WSL~/.local/bin/claude
Windows%USERPROFILE%\.local\bin\claude

設定ファイル

ファイル役割
CLAUDE.mdAI への常時指示書
~/.claude/settings.jsonユーザー設定
.claude/settings.jsonプロジェクト固有設定
~/.claude.json認証情報・MCP
.mcp.jsonMCP サーバー定義

PATH が通っていない場合

command not found が出たら実行してください。

Mac / Linux（zsh）:

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

Windows PowerShell:

$p = [Environment]::GetEnvironmentVariable('PATH','User')
[Environment]::SetEnvironmentVariable('PATH',"$p;$env:USERPROFILE\.local\bin",'User')

→ PowerShell を再起動すると反映。

トラブルシューティング 共通

「claude が認識されません」

1

ターミナルを閉じて開き直す（90%解決）

2

Cursor を完全再起動

3

PATH を手動で追加（上のセクション参照）

「Git for Windows required」

1

git-scm.com → Download → インストール

2

Cursor を再起動（しないと認識されない）

3

インストールコマンドを再実行

「irm が認識されません」

cmd を使っている → PowerShell に切り替え（▼ → PowerShell）、または cmd 用コマンドを使う。

ブラウザが開かない

ターミナルに表示された URL をコピー → ブラウザに手動で貼り付け。

TLS / ネットワークエラー

VPN OFF → Wi-Fi 切替 → テザリング → 講師に相談。

最終手段

claude doctor
自動診断コマンド。結果をスクリーンショットして講師に共有してください。

完了チェックリスト

  • claude --version でバージョンが表示される
  • claude doctor で全て通る
  • claude → /status でアカウント情報が出る
  • 「自己紹介して」と入力して回答が返ってくる
