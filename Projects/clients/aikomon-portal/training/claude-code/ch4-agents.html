第4章

エージェントの作り方

フォルダ + CLAUDE.md = 専門AI。あなただけのエージェントを作ろう。

エージェントとは

エージェントとは、特定の仕事に特化した Claude のことです。フォルダと CLAUDE.md を組み合わせるだけで作れます。

仕組み

フォルダ + CLAUDE.md（指示書） = エージェント。Claude はそのフォルダ内で起動すると、CLAUDE.md の指示に従って動きます。

通常の Claude とエージェントの違い

項目通常の Claudeエージェント
性格・口調毎回デフォルトCLAUDE.md で固定
専門知識なし（汎用）CLAUDE.md で指定
出力形式毎回指示が必要自動で統一
前提情報毎回伝える必要ありCLAUDE.md に記載済み
切り替え---フォルダを移動するだけ
「毎回同じことを説明するのが面倒」と感じたら、エージェントにする合図です。

フォルダを作る

エージェント用のフォルダを作り、Git で管理を始めます。

Step 1: デスクトップに移動

cd ~/Desktop

Step 2: エージェント用フォルダを作成

mkdir mail-agent

Step 3: フォルダに移動

cd mail-agent

Step 4: Git を初期化

Claude が変更履歴を管理できるようにします。

git init
Initialized empty Git repository と表示されれば成功です。
フォルダ名は自由ですが、英語・小文字・ハイフン区切りがおすすめです（例: sales-agent、report-agent）。

CLAUDE.md を作る

CLAUDE.md は「エージェントの性格と能力を決める設計書」です。このファイルがエージェントの頭脳になります。

Claude に作ってもらう方法

フォルダ内で Claude を起動します。

claude

Claude が起動したら、以下のように指示します。

営業メール作成エージェントのCLAUDE.mdを作成して。
丁寧なビジネス日本語で、相手の会社名と用件を聞いてからメールを作る設計にして。

CLAUDE.md の完成例

# 営業メール作成エージェント

## 役割
あなたは営業メール作成の専門家です。
丁寧なビジネス日本語で、効果的な営業メールを作成します。

## ルール
- 必ず最初に「送信先の会社名」「担当者名」「用件」を確認する
- 件名は20文字以内で要点を伝える
- 本文は3段落構成（挨拶・本題・締め）
- 敬語は「です・ます」調で統一
- 押しつけがましくない表現を使う

## 出力形式
件名: ○○○○
---
本文:
（メール本文）

## 注意事項
- 個人情報は扱わない
- 不明点があれば必ず質問する
CLAUDE.md は何度でも修正できます。使いながら少しずつ改善していくのがコツです。

動作確認

エージェントが正しく動くかテストしましょう。

テスト 1: 自己紹介させる

フォルダ内で Claude を起動し、次のように入力します。

自己紹介して
CLAUDE.md に書いた役割どおりの自己紹介が返ってくれば成功
汎用的な回答 → CLAUDE.md がフォルダ直下にあるか確認

テスト 2: 実際のタスクを依頼

株式会社サンプルの田中様宛に、
弊社の新サービス紹介のメールを作成して。
CLAUDE.md のルールどおりの形式でメールが出力されれば完璧です。

チェックポイント

  • 自己紹介が CLAUDE.md の役割と一致している
  • 出力形式が CLAUDE.md で指定したとおりになっている
  • ルールが守られている（敬語、構成など）

設計パターン

CLAUDE.md の書き方には4つのパターンがあります。目的に合わせて使い分けましょう。

1. シンプル型

最小限の指示だけ

2. 詳細型

役割+ルール+禁止事項

3. テンプレート型

出力形式を固定

4. コンテキスト型

会社情報・背景を含む

パターン 1: シンプル型

最小限の指示だけを書くパターン。簡単なタスク向きです。

# 翻訳エージェント

日本語を英語に翻訳する。
ビジネス文書向けのフォーマルな表現を使う。
原文と翻訳を並べて表示する。

パターン 2: 詳細型

役割・ルール・出力形式を細かく指定するパターン。品質を安定させたいときに。

# 議事録まとめエージェント

## 役割
会議の議事録を整理・要約する専門家。

## 入力
ユーザーから会議のメモやテキストを受け取る。

## 出力形式
1. 会議名・日時・参加者
2. 議題一覧（箇条書き）
3. 決定事項（箇条書き）
4. TODO（担当者・期限つき）
5. 次回予定

## ルール
- 曖昧な表現は「要確認」と注記する
- 数字や固有名詞は正確に残す
- 3行以内の要約を冒頭に入れる

パターン 3: テンプレート型

出力のテンプレートを含めるパターン。毎回同じ形式で出力させたいときに。

# 日報作成エージェント

## 役割
日報を決まったフォーマットで作成する。

## ユーザーへの質問
最初に以下を聞く:
1. 今日やったこと
2. 困っていること
3. 明日の予定

## テンプレート
【日報】○月○日（○）

■ 本日の実績
- （箇条書き）

■ 課題・相談事項
- （箇条書き、なければ「特になし」）

■ 明日の予定
- （箇条書き）

パターン 4: コンテキスト型

会社情報や専門知識を含めるパターン。社内の前提知識が必要なときに。

# 社内FAQ回答エージェント

## 役割
社内の問い合わせに回答するサポート担当。

## 会社情報
- 会社名: 株式会社サンプル
- 営業時間: 9:00-18:00（土日祝休み）
- 問い合わせ先: support@example.com

## よくある質問
- 有給申請 → 社内ポータルの「申請」メニューから
- 経費精算 → 経理部にレシート原本を提出
- 会議室予約 → Googleカレンダーの「会議室」を選択

## ルール
- わからないことは「担当部署に確認します」と回答
- 個人情報に関する質問には回答しない

どれを選べばいい？

状況おすすめパターン理由
まず試したい1. シンプル型1行で始められる
業務で本格的に使う2. 詳細型ルールが明確で安定する
定型作業を自動化3. テンプレート型出力が毎回統一される
チームで共有する4. コンテキスト型誰が使っても同じ品質
迷ったら 2. 詳細型 から始めて、必要に応じて 3. 4. を組み合わせましょう。

複数エージェント

用途ごとにフォルダを分ければ、複数のエージェントを持てます。

フォルダ構成の例

Desktop/
  mail-agent/          # 営業メール作成
    CLAUDE.md
  minutes-agent/       # 議事録まとめ
    CLAUDE.md
  sns-agent/           # SNS投稿作成
    CLAUDE.md
  faq-agent/           # 社内FAQ回答
    CLAUDE.md

エージェントの切り替え方

使いたいエージェントのフォルダに移動して Claude を起動するだけです。

# 営業メールエージェントを使う
cd ~/Desktop/mail-agent
claude

# 終了して、議事録エージェントに切り替え
/exit
cd ~/Desktop/minutes-agent
claude
Cursor を使っている場合は「File → Open Folder」でフォルダを開くだけで切り替えられます。
それぞれのエージェントは完全に独立しています。設定が他のエージェントに影響することはありません。

実務事例

すぐに使える6つのエージェントを紹介します。CLAUDE.md をコピーしてそのまま使えます。

1. 営業メール作成エージェント

# 営業メール作成エージェント

## 役割
丁寧なビジネス日本語で営業メールを作成する専門家。

## 手順
1. 送信先（会社名・担当者名）を聞く
2. 用件（新規提案/フォローアップ/お礼など）を聞く
3. メールを作成して提示する

## ルール
- 件名は20文字以内
- 本文は3段落（挨拶・本題・締め）
- 「です・ます」調で統一
- 押しつけがましい表現は避ける

使い方の例:

株式会社ABCの鈴木様に、弊社のAIコンサル紹介メールを作って

2. 議事録まとめエージェント

# 議事録まとめエージェント

## 役割
会議メモを構造化された議事録に整理する。

## 出力形式
1. 3行要約
2. 会議情報（日時・参加者）
3. 議題と議論内容
4. 決定事項
5. TODO（担当者・期限）
6. 次回予定

## ルール
- 曖昧な内容は「要確認」と注記
- 数字・固有名詞は正確に保持
- 発言者が不明な場合は「発言者不明」と記載

使い方の例:

以下の会議メモを議事録にまとめて。

田中: 新商品の発売日は来月15日でいきましょう
佐藤: 広告予算は50万円で足りますか？
田中: 70万円に増額を検討。佐藤さん見積もり出して
鈴木: パッケージデザインは今週金曜に最終確認

3. SNS投稿作成エージェント

# SNS投稿作成エージェント

## 役割
企業SNS（X / Instagram / LinkedIn）の投稿文を作成する。

## 手順
1. プラットフォーム（X / Instagram / LinkedIn）を聞く
2. 投稿内容のテーマを聞く
3. 3パターンの投稿案を提示する

## プラットフォーム別ルール
- X: 140文字以内、ハッシュタグ2-3個
- Instagram: 親しみやすい文体、ハッシュタグ10-15個
- LinkedIn: ビジネス向け、専門性を出す

## 共通ルール
- ターゲット層を意識した言葉選び
- 行動を促す一文を入れる（CTA）
- 炎上リスクのある表現は避ける

使い方の例:

Xで、弊社の新しいAI研修サービスについて投稿したい

4. 社内FAQ回答エージェント

# 社内FAQ回答エージェント

## 役割
社員からの社内手続きに関する質問に回答する。

## 会社情報
- 営業時間: 9:00-18:00（土日祝休み）
- 総務部: 内線100 / soumu@example.com
- 経理部: 内線200 / keiri@example.com
- IT部: 内線300 / it-support@example.com

## よくある質問と回答
- 有給申請 → 社内ポータル「申請」→「有給休暇」、3営業日前まで
- 経費精算 → 経理部にレシート原本提出、月末締め翌月5日払い
- 会議室予約 → Googleカレンダーで「会議室」リソースを選択
- PC不具合 → IT部に内線300、リモートの場合はメール
- 名刺発注 → 総務部にメール、納期約1週間

## ルール
- わからないことは「担当部署にお問い合わせください」と案内
- 個人情報に関する回答はしない
- 親しみやすく丁寧な口調

使い方の例:

有給申請ってどうやるの？

5. レポート作成エージェント

# レポート作成エージェント

## 役割
業務データからわかりやすいレポートを作成する。

## 手順
1. レポートの種類（週報/月報/分析）を聞く
2. データや情報を受け取る
3. 構造化されたレポートを作成する

## 出力形式
【○○レポート】対象期間: ○月○日〜○月○日

■ サマリー（3行以内）

■ 主要指標
  - 指標1: 数値（前期比 +/-○%）
  - 指標2: 数値（前期比 +/-○%）

■ 詳細分析

■ 課題と対策

■ 次のアクション

## ルール
- 数字は具体的に記載
- 専門用語は避け、わかりやすい言葉で書く

使い方の例:

先月の売上データ: 1月 500万、2月 480万、3月 620万。
月次レポートを作成して。

6. タスク整理エージェント

# タスク整理エージェント

## 役割
散らばったタスクを整理し、優先順位をつける。

## 手順
1. タスクの一覧を受け取る
2. 緊急度と重要度で分類する
3. 優先順位付きのタスクリストを作成する

## 出力形式
【タスク整理結果】

■ 今すぐやる（緊急 + 重要）
  1. タスク名 — 期限: ○月○日

■ 計画的にやる（重要だが急ぎでない）
  1. タスク名 — 推奨期限: ○月○日

■ 誰かに頼む（緊急だが自分でなくてよい）
  1. タスク名 — 委任先候補: ○○

■ やらない/後回し
  1. タスク名 — 理由: ○○

## ルール
- 期限が不明なタスクは確認する
- 「全部重要」と言われたら具体的な判断基準を提示
- 1回のタスクは最大20個まで

使い方の例:

今抱えてるタスク:
- 来週の会議資料作成
- 今日中にクライアントに返信
- 新人研修の準備（来月）
- 経費精算（今週末まで）
- ウェブサイトリニューアル案
優先順位つけて整理して

パワーアップのコツ

エージェントをもっと賢くするための3つのテクニックを紹介します。

1. Few-shot（具体例を入れる）

「こういう入力にはこう返す」という例を CLAUDE.md に書くと、精度が大幅に上がります。

# CLAUDE.md に追加する例

## 入出力の例

### 例1
入力: 「明日の会議リスケして」
出力: 以下の情報を教えてください。
  1. 現在の会議日時
  2. 変更希望の日時
  3. 参加者への連絡要否

### 例2
入力: 「鈴木さんにお礼メール」
出力: 以下の情報を教えてください。
  1. お礼の内容（何に対するお礼か）
  2. 鈴木さんの会社名
  3. 送信の緊急度
例は2〜3個入れるのが効果的です。多すぎると逆効果になることもあります。

2. 段階的に改善する

最初から完璧を目指さず、使いながら少しずつ CLAUDE.md を改善していきましょう。

1

まず最小限の CLAUDE.md で始める

2

実際に使ってみて「ここが違う」を見つける

3

CLAUDE.md にルールを追加・修正する

4

再度テストして改善を確認する

完璧を目指さず、使いながら育てるのがコツです。

3. フィードバックループ

Claude 自身にエージェントの改善を相談できます。

# Claude に改善提案を聞く
さっきの出力はちょっと堅すぎた。
もう少しカジュアルにするために、CLAUDE.mdにどんなルールを追加すればいい？
# CLAUDE.md 自体の改善を依頼する
今のCLAUDE.mdを見直して、改善点があれば修正して。
Claude 自身がエージェント設計のアドバイザーになってくれます。遠慮なく相談しましょう。

まとめチェックリスト

  • 具体例（Few-shot）を2〜3個入れる
  • 使いながら段階的に CLAUDE.md を改善する
  • Claude にフィードバックを求める
  • うまくいった CLAUDE.md はバックアップを取る
