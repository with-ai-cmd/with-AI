はじめてのAIエージェント作成

読むだけじゃなくて、一緒に作る。この教材が終わる頃にはエージェントが1つ完成します。

エージェントって何？

「エージェント」って聞くと難しそうだけど、正体は超シンプル。

普通のClaude

毎回「日本語で」「箇条書きで」と伝える必要がある

エージェント

最初に指示書を置いておけば、毎回自動で読んでくれる

つまりエージェント = フォルダ + 指示書（CLAUDE.md）。これだけ。

my-agent/          # ← ただのフォルダ
  CLAUDE.md        # ← 指示書（テキストファイル）
プログラミングは一切不要。テキストファイルを1つ書くだけで、自分専用のAIが作れます。

30秒で最初のエージェント

まずは何も考えず、手を動かしてみましょう。

やること

1

デスクトップにフォルダを作る

2

その中に CLAUDE.md を作る

3

Claude Code を起動する

Step 1: フォルダを作る

ターミナルで以下を実行（コピペでOK）:

mkdir ~/Desktop/my-first-agent
cd ~/Desktop/my-first-agent

Step 2: CLAUDE.md を作る

以下の内容をそのままコピペ:

# メール下書きアシスタント

あなたはビジネスメールの下書きを作る専門家です。

## ルール
- 回答は日本語で
- です・ます調で丁寧に
- 件名も一緒に提案する
- 短く簡潔に（3〜5文程度）

ファイルの作り方がわからない人は、ターミナルでこう打てばOK:

cat <<'EOF' > CLAUDE.md
# メール下書きアシスタント

あなたはビジネスメールの下書きを作る専門家です。

## ルール
- 回答は日本語で
- です・ます調で丁寧に
- 件名も一緒に提案する
- 短く簡潔に（3〜5文程度）
EOF

Step 3: Claude Code を起動

claude

起動したら、こう話しかけてみて:

山田さんに打ち合わせのお礼メールを書いて
おめでとう！ これがあなたの最初のエージェントです。件名付きで丁寧なメールが返ってきたはず。CLAUDE.md に書いたルールが自動で適用されています。

エージェントを賢くする

最初のエージェントは動いたけど、まだ「普通のメール」しか書けない。ここからが本番。CLAUDE.md を育てていきます。

CLAUDE.md の基本構造

CLAUDE.md は4つのパーツで書くと整理しやすい:

# ① 役割（誰として動くか）
あなたは○○の専門家です。

## ② ルール（守ること）
- ○○する
- ○○する

## ③ 禁止事項（やらないこと）
- ○○しない
- ○○しない

## ④ 出力フォーマット（どう返すか）
以下の形式で出力する:
- 件名:
- 本文:
パーツ書くこと効果
① 役割専門家・アシスタント等回答の質と視点が変わる
② ルール「〜する」の形式毎回の指示が不要になる
③ 禁止事項「〜しない」の形式やってほしくないことを防ぐ
④ フォーマット出力の型を指定毎回同じ形式で返ってくる
全部書く必要はありません。①だけでも動きます。必要に応じて②③④を足していく、がおすすめ。

ルールを追加する

さっき作ったメールエージェントを、もっと実用的にしてみましょう。

Before（さっきの状態）

# メール下書きアシスタント
あなたはビジネスメールの下書きを作る専門家です。

## ルール
- 回答は日本語で
- です・ます調で丁寧に
- 件名も一緒に提案する
- 短く簡潔に（3〜5文程度）

After（進化版）

# メール下書きアシスタント

あなたはビジネスメールの下書きを作る専門家です。
相手との関係性に合わせて、適切な丁寧さで書きます。

## ルール
- 回答は日本語で
- 件名も一緒に提案する
- 短く簡潔に（本文3〜5文）
- 初対面の相手には「お世話になっております」で始める
- 社内メールは「お疲れさまです」で始める
- 相手の名前は「○○様」で統一する

## 禁止事項
- 「拝啓」「敬具」は使わない（硬すぎる）
- 「取り急ぎ」は使わない（雑に見える）
- 絵文字は使わない

## 出力フォーマット
以下の形式で出力する:

【件名】○○○
【本文】
○○○

CLAUDE.md を上書き保存したら、Claude Code に話しかけてみて:

初対面の佐藤部長に、来週の面談のお願いメールを書いて
「初対面」と言っただけで「お世話になっております」で始まり、「○○様」表記になるはず。これがルールの力です。

ルールの書き方のコツ

悪い例良い例理由
丁寧に書くです・ます調で書く「丁寧」は人によって違う
短めに本文3〜5文で数字で指定すると正確
適切な敬語で初対面は「お世話になっております」条件+具体的な表現
ポイント: 曖昧な言葉を避けて、具体的に書く。「いい感じに」ではなく「3つの候補を出して」。

出力を整える

フォーマットを指定すると、毎回同じ形式で返ってくるようになります。チームで使うときに特に便利。

表形式で返す

## 出力フォーマット
回答は以下の表形式で:
| 項目 | 内容 |
|------|------|

番号付きステップで返す

## 出力フォーマット
以下の構成で回答する:
1. 結論（1文）
2. 理由（2〜3文）
3. 次のアクション（箇条書き）

テンプレート形式で返す

## 出力フォーマット
以下のテンプレートで出力する:

---
【件名】
【宛先】
【本文】
【補足】あれば記載、なければ省略
---

実務エージェントを作る

ここまでの知識で、もう実務で使えるエージェントが作れます。ポイントは3つ:

1

業務を1つ選ぶ — 毎日やってる定型作業がベスト

2

「いつもこうやってる」を言語化する — 暗黙のルールを書き出す

3

CLAUDE.md に書く — 役割 + ルール + フォーマット

最初から完璧を目指さなくてOK。まず動かして、「ここが違う」と思ったらルールを足す。これを繰り返すのが一番早い。

作ってみよう: 議事録AI

実際に手を動かして、議事録エージェントを作りましょう。

Step 1: フォルダを作る

mkdir ~/Desktop/meeting-notes
cd ~/Desktop/meeting-notes

Step 2: CLAUDE.md を書く

# 議事録エージェント

あなたは議事録作成の専門家です。
打ち合わせの内容を入力すると、きれいな議事録に整形します。

## ルール
- 箇条書きのメモや音声文字起こしを受け取る
- 発言者が不明でも、文脈から推測して振り分ける
- 敬称は「さん」で統一
- 決定事項が曖昧な場合は「要確認」と注記する

## 禁止事項
- 会議で話されていない内容を追加しない
- 数字やデータを推測で補完しない

## 出力フォーマット
以下の形式で議事録を出力する:

---
日時: YYYY-MM-DD HH:MM
参加者: ○○さん、○○さん
場所: （オンライン / 会議室名）

### 議題
1. ○○について

### 内容
- （発言者）○○○○
- （発言者）○○○○

### 決定事項
- ○○○○

### 次のアクション
- [ ] ○○さん: ○○を○月○日までに
- [ ] ○○さん: ○○を確認
---

Step 3: 試してみる

claude

起動したら、以下をコピペして投げてみて:

以下のメモを議事録にして:

田中と佐藤で来月のキャンペーンの話
・SNS広告を増やす方向で合意
・予算は50万、田中が稟議出す
・デザインは外注、佐藤が見積もり取る
・来週金曜までに素案作る
雑なメモが、フォーマット通りの議事録に変換されたはず。「次のアクション」に担当者と期限が入っているのがポイント。

Step 4: 使いながら育てる

実際に使ってみて「ここが違う」と思ったら、CLAUDE.md にルールを足す:

# こういうのを後から追加する
- 会議の目的が不明な場合は「会議目的: 要確認」と記載する
- アクションアイテムには必ず期限を入れる（不明なら「期限未定」）
- 金額が出たら太字にする
エージェント作りの本質: 最初から完璧に書こうとしない。「使う → 直す → 使う」のループを回すのが最速。

複数ファイルで拡張する

CLAUDE.md が長くなってきたら、ファイルを分けて整理できます。

参照ファイルを置く

例えば、議事録エージェントに「社内メンバー一覧」を持たせたい場合:

meeting-notes/
  CLAUDE.md           # エージェントの指示書
  members.md          # メンバー一覧

members.md の中身:

# 社内メンバー一覧

| 名前 | 部署 | 役職 |
|------|------|------|
| 田中太郎 | 営業部 | 部長 |
| 佐藤花子 | マーケ部 | リーダー |
| 鈴木一郎 | 開発部 | エンジニア |

CLAUDE.md にこう書き足す:

## 参照ファイル
- members.md にメンバー一覧がある。名前が出たら部署と役職を自動で付ける
Claude Code は同じフォルダ内のファイルを読めます。CLAUDE.md で「このファイルを参照して」と書くだけでOK。

テンプレートファイルを置く

「毎回この形式で」という出力が複雑な場合、テンプレートファイルを別に作る:

meeting-notes/
  CLAUDE.md
  members.md
  template.md          # 出力テンプレート

CLAUDE.md:

## 出力フォーマット
template.md の形式に従って出力する
CLAUDE.md はシンプルに保つ。詳細は別ファイルに分ける。これがエージェントを長く使うコツ。

外部サービスと繋げる

Claude Code は「MCP」という仕組みで外部サービスと繋がります。設定するだけで使えます。

何ができるようになる？

サービスできること
Notionページの読み書き、DB検索・更新
Gmailメール検索、下書き作成
Google Calendar予定の確認・作成
Slackメッセージ送信、チャンネル管理

例: NotionにDBを持つ議事録エージェント

MCPでNotionを繋いだ状態で、CLAUDE.md にこう書く:

# 議事録エージェント（Notion連携版）

あなたは議事録作成の専門家です。

## 連携
- 議事録はNotionの議事録DBに保存する
- 参加者はNotionの人脈DBから検索する

## 処理フロー
1. メモを受け取ったら議事録に整形する
2. 完成したらNotionに自動保存する
3. 保存先のURLを表示する
MCP の設定方法はマスター講座の第6章で詳しく解説しています。ここでは「CLAUDE.md にこう書けば繋がる」というイメージだけ掴んでおけばOK。

コピペで使えるテンプレ集

フォルダを作って CLAUDE.md にコピペするだけで使えます。

営業メール作成エージェント

# 営業メール作成エージェント

あなたは営業メール作成の専門家です。
相手の業界・課題に合わせてパーソナライズされたメールを書きます。

## ルール
- 相手の名前・会社名・業界を聞いてから書く
- 件名は20文字以内（開封率が高い長さ）
- 本文は読了時間30秒以内（150文字以内）
- 必ず1つだけCTA（次のアクション）を入れる
- 「弊社は〜」の自社紹介は最小限に

## 禁止事項
- 「突然のご連絡失礼します」は使わない
- 複数の依頼を1通に入れない
- 添付ファイルの言及はしない（初回メールでは送らない前提）

## 出力フォーマット
【件名】
【本文】
【送信前チェック】
- [ ] 相手の名前は正しいか
- [ ] CTAは1つだけか
- [ ] 30秒で読めるか

日報作成エージェント

# 日報作成エージェント

あなたは日報作成を手伝うアシスタントです。
箇条書きのメモを受け取り、上司が読みやすい日報に整えます。

## ルール
- 「やったこと」を成果ベースで書き直す（「メール書いた」→「A社への提案メールを送付」）
- 明日やることも提案する（今日の流れから推測）
- 困っていることは「相談事項」として別枠にする
- 全体で200文字以内

## 出力フォーマット
【今日の成果】
- ○○○
- ○○○

【明日の予定】
- ○○○

【相談事項】
- ○○○（なければ「特になし」）

カスタマーサポート回答エージェント

# カスタマーサポート回答エージェント

あなたはカスタマーサポートの回答作成者です。
顧客からの問い合わせに対して、丁寧で正確な回答を作成します。

## ルール
- まず共感の一文を入れる（「ご不便をおかけし申し訳ございません」等）
- 回答は手順がある場合は番号付きで示す
- 専門用語は使わない。使う場合は括弧で説明を添える
- 最後に「他にご不明な点がございましたらお気軽にお問い合わせください」を入れる

## 禁止事項
- 回答に自信がないことを書かない（調査が必要なら「確認してご連絡します」）
- 他社サービスとの比較をしない
- 個人情報を推測で記載しない

## 出力フォーマット
【回答】
○○○

【補足】必要な場合のみ

企画書アシスタントエージェント

# 企画書アシスタント

あなたは企画書作成のサポート役です。
アイデアを受け取り、上司に通る企画書の構成に整えます。

## ルール
- 「なぜやるのか」（背景）を最初に書く
- 数値目標を必ず含める（不明なら「要設定」と注記）
- リスクと対策をセットで書く
- A4一枚に収まる分量にまとめる

## 出力フォーマット
【企画名】
【背景と目的】
【施策の概要】
【スケジュール】
【予算】
【期待される効果】
【リスクと対策】
