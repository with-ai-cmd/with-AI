# Jungle Gym Safari — 技術アーキテクチャ

## 設計方針

**「作らない」が最強のコスト削減。**
ライブ配信・EC・決済は既存プラットフォーム（YouTube Live / Shopify）に乗せ、
自前で作るのはコミュニティ体験のコア部分（会員管理・コンテンツ制御・タイムライン・特典）に絞る。

## 自前構築 vs 外部サービス 判断表

| 機能 | 方針 | サービス | 理由 |
|---|---|---|---|
| ライブ配信 | 外部 | YouTube Live | 配信基盤・チャット・アーカイブが全部無料。開発コストゼロ |
| EC・物販 | 外部 | Shopify | EC構築不要。在庫・注文・配送が全部ある |
| チケット販売 | 外部 | Shopify（デジタル商品） | チケット = Shopifyのデジタル商品として販売 |
| 決済（EC） | 外部 | Shopify Payments | クレカ・PayPay対応済み。自前決済連携不要 |
| 決済（サブスク） | 外部 | Stripe Billing | サブスク管理の定番。Shopifyとは別管理 |
| コンテンツ課金 | 外部 | Stripe | 個別コンテンツの都度課金 |
| 会員管理・認証 | **自前** | NestJS + Supabase | プラン制御のコア。外部に任せられない |
| コンテンツ配信 | **自前** | S3 + CloudFront | プラン別アクセス制御が必要 |
| タイムライン | **自前** | NestJS | コミュニティのコア体験 |
| 会員特典 | **自前** | NestJS | QRコード会員証、利用管理 |
| プッシュ通知 | 外部 | Firebase (FCM) | 標準的で無料 |
| 管理画面 | **自前** | Next.js | 各サービスの統合ダッシュボード |

## 技術スタック

### フロントエンド

| 領域 | 技術 | 理由 |
|---|---|---|
| Web | Next.js 15 (App Router) | SSR/SSG対応、SEO、パフォーマンス |
| モバイル | React Native (Expo) | Webとコード共有可能、開発速度 |
| UI | Tailwind CSS + shadcn/ui | 西海岸テイストのカスタムデザインに柔軟 |
| 状態管理 | TanStack Query | サーバーステート管理 |
| 動画プレーヤー | YouTube IFrame API | YouTube Live/アーカイブの埋め込み |

### バックエンド

| 領域 | 技術 | 理由 |
|---|---|---|
| API Server | NestJS (TypeScript) | モジュラー設計、スケーラブル |
| DB | Supabase (PostgreSQL) | 認証・DB・ストレージ一体。コスト低い |
| Cache | Redis (Upstash) | サーバーレスRedis。タイムラインキャッシュ |
| Storage | S3 + CloudFront | 動画/画像配信、CDN |

### インフラ

| 領域 | 技術 | 理由 |
|---|---|---|
| ホスティング（Web） | Vercel | Next.jsと相性最高。デプロイ自動化 |
| ホスティング（API） | Railway or Render | NestJSのデプロイが簡単。初期コスト低い |
| CI/CD | GitHub Actions | 開発フロー統合 |
| モニタリング | Sentry | エラー追跡 |

## YouTube Live 連携設計

### 配信フロー

```
運営側:
  1. YouTube Studio でライブ配信を作成
  2. 管理画面で配信情報を登録（YouTube動画ID + プラン制限 + アーカイブフラグ）
  3. 配信開始 → アプリ側にプッシュ通知

ユーザー側:
  1. プッシュ通知 or アプリ内で配信を確認
  2. プランチェック → 視聴権限があれば YouTube Player を表示
  3. チャットは YouTube Live のチャットをそのまま利用

アーカイブ:
  - 「残す」→ YouTube側でアーカイブ公開 → アプリにアーカイブとして表示
  - 「残さない」→ YouTube側で非公開/削除 → アプリから非表示
```

### プラン別制限の仕組み

```
YouTube側: 全配信を「限定公開」で作成（URLを知っている人のみ視聴可能）

アプリ側:
  - 管理画面で各配信に「視聴可能プラン」を設定
  - ユーザーのプランを確認
  - 権限がある場合のみ YouTube IFrame を表示（限定公開URLを渡す）
  - 権限がない場合 → アップグレード案内を表示

※ YouTube URLの直接共有リスクはあるが、会員コミュニティの性質上
   意図的な流出は低リスク。必要なら定期的にURL再生成で対応。
```

### YouTube Data API v3 の利用

| 用途 | API | 備考 |
|---|---|---|
| 配信情報取得 | liveBroadcasts.list | 配信ステータス・視聴者数 |
| アーカイブ管理 | videos.update | 公開/非公開の切り替え |
| チャット取得 | liveChatMessages.list | 必要に応じてアプリ内表示 |

## Shopify 連携設計

### ECフロー

```
Shopify側:
  - 商品登録（アパレル、グッズ、サプリ等）
  - デジタル商品としてイベントチケット登録
  - Shopify Payments で決済（クレカ / PayPay）
  - 在庫・注文・配送の管理

アプリ側（Storefront API 経由）:
  - 商品一覧をアプリ内に表示
  - アプリ内からShopify Checkoutに遷移して購入
  - 注文履歴の表示
  - プラン別割引 → Shopify のディスカウントコード or 会員限定コレクション
```

### チケット販売

```
仕組み:
  1. イベントチケット = Shopify のデジタル商品
  2. 購入完了 → メールでチケット（QRコード）送信
  3. イベント当日 → アプリの会員証 or チケットQRを提示

YouTube Live チケット配信:
  1. 有料イベント = Shopifyでチケット販売
  2. 購入者にYouTube限定公開URLを配信（メール or アプリ内通知）
  3. 購入者のみ視聴可能
```

### Storefront API で取得するデータ

| データ | 用途 |
|---|---|
| 商品一覧 | アプリ内EC画面 |
| 商品詳細（画像・価格・在庫） | 商品詳細画面 |
| コレクション | カテゴリ分け・会員限定商品 |
| チェックアウトURL | 購入フローへの遷移 |
| 注文履歴 | マイページの購入履歴 |

## Stripe 連携設計（サブスク + コンテンツ課金）

### サブスクリプション

```
Stripe Billing:
  - 3プラン（Bronze / Silver / Gold）を Stripe Products として登録
  - 月額自動課金
  - Customer Portal でプラン変更・解約をユーザー自身で操作
  - Webhook でプラン変更を検知 → アプリ側の権限を即時更新
```

### コンテンツ個別課金

```
Stripe Checkout:
  - Bronze/Silver ユーザーが追加カテゴリを購入
  - 1回の決済でカテゴリアクセス権を付与
  - 月額追加 or 買い切り（要確認）
```

### 決済の棲み分け

| 対象 | 決済サービス | 理由 |
|---|---|---|
| サブスク月額 | Stripe Billing | サブスク管理に最適化 |
| コンテンツ追加課金 | Stripe Checkout | 都度課金 |
| 物販・グッズ | Shopify Payments | EC内で完結 |
| イベントチケット | Shopify Payments | デジタル商品として販売 |

## モジュール設計（自前構築分のみ）

### Auth Module
- Supabase Auth（メール + ソーシャルログイン）
- Stripe Customer ID との紐付け
- プラン権限管理（RBAC）

### Content Module
- 動画アップロード → S3 → CloudFront 配信
- カテゴリ管理（5カテゴリ）
- プラン別アクセス制御（署名付きURL）
- 視聴履歴・進捗トラッキング

### Timeline Module
- 投稿CRUD（テキスト + メディア）
- Instagram API 連携（投稿取り込み）
- フィード生成（タブ分け対応）
- いいね・コメント

### Perks Module（会員特典）
- 特典マスタ管理
- デジタル会員証（QRコード）
- クーポン発行・利用管理
- 利用履歴

### Notification Module
- FCM（Firebase Cloud Messaging）/ APNs
- ユーザー別通知設定
- セグメント配信
- 配信スケジュール管理

### Admin Module
- ダッシュボード（会員数、売上、アクティブ率）
- YouTube Live 配信管理（動画ID登録、プラン制限設定）
- Shopify 売上連携表示
- Stripe サブスク状況表示
- 各モジュールの管理CRUD
