## 3. 機能詳細

### F-01: アカウント管理

```
登録情報（想定）:
  - 名前（氏名）
  - 生年月日
  - メールアドレス
  - パスワード
  ※ その他取得項目はヒアリングシートで確認

ログイン方法:
  - メール + パスワード
  - ソーシャルログイン（Apple / Google — 要確認）

機能:
  - プロフィール編集（アイコン、自己紹介）
  - プラン情報表示
  - デジタル会員証表示
```

### F-02: サブスクリプション（3プラン）

```
プラン構成:
  ┌──────────────────────────────────────────────────────┐
  │ プラン1（低価格帯）                                     │
  │  - コンテンツ: 5カテゴリから1つ無料で選択               │
  │  - 追加コンテンツは個別課金                             │
  │  - 基本的な会員特典                                    │
  └──────────────────────────────────────────────────────┘
  ┌──────────────────────────────────────────────────────┐
  │ プラン2（中価格帯）                                     │
  │  - コンテンツ: 5カテゴリから3つ無料で選択               │
  │  - 追加コンテンツは個別課金                             │
  │  - 中位の会員特典                                      │
  └──────────────────────────────────────────────────────┘
  ┌──────────────────────────────────────────────────────┐
  │ プラン3（高価格帯）                                     │
  │  - コンテンツ: 全5カテゴリ見放題                        │
  │  - 追加課金なし                                        │
  │  - 全会員特典利用可能                                  │
  └──────────────────────────────────────────────────────┘

※ プラン名称・価格はクライアント確認待ち
```

### F-03: 決済（サブスク・コンテンツ課金 → Stripe）

```
Stripe Billing:
  - 3プラン（Bronze / Silver / Gold）を月額自動課金
  - Customer Portal でプラン変更・解約をユーザー自身で操作
  - Webhook でプラン変更を検知 → アプリ側の権限を即時更新

Stripe Checkout:
  - Bronze/Silver ユーザーの追加カテゴリ購入（都度課金）
```

### F-04: 決済（EC・チケット → Shopify Payments）

```
Shopify Payments:
  - クレジットカード（Visa / Master / AMEX / JCB）
  - PayPay
  - 物販・グッズ・イベントチケットの決済はShopify内で完結
  - 自前で決済連携を実装する必要なし

Phase 2:
  - 口座引き落とし（要確認）
```

### F-05/06: コンテンツ配信・課金

```
カテゴリ（5つ）:
  1. 美容 — スキンケア、メイク、ヘアケア
  2. 自宅トレーニング — ワークアウト、食事管理
  3. 金融 — 投資、資産運用、マネーリテラシー
  4. 健康 — メンタルヘルス、栄養学、ウェルネス
  5. スタートアップ — 起業、ビジネス戦略

形式:
  - 動画（メインコンテンツ）
  - スライド / PDF（補助教材）

プラン別アクセス:
  - 各プランで無料アクセスできるカテゴリ数が異なる
  - 無料枠外のカテゴリは個別課金で解放
  - Gold プランは全カテゴリ無制限
```

### F-07: ライブ配信（YouTube Live 活用）

```
基盤: YouTube Live（自前構築しない → コスト大幅削減）

配信フロー:
  1. 運営が YouTube Studio でライブ配信を作成（限定公開）
  2. 管理画面で配信情報を登録（YouTube動画ID + プラン制限 + アーカイブフラグ）
  3. 配信開始 → アプリにプッシュ通知
  4. ユーザーがアプリ内でYouTube Playerを通じて視聴

チャット: YouTube Live のチャット機能をそのまま利用

アーカイブ制御:
  - 「残す」→ YouTube側でアーカイブ公開 → アプリにアーカイブとして表示
  - 「残さない」→ YouTube側で非公開/削除 → アプリから非表示

プラン別制限:
  - 全配信を YouTube「限定公開」で作成
  - アプリ側でプランチェック → 権限があればYouTube Player表示
  - 権限がなければアップグレード案内を表示

有料イベント配信:
  - Shopifyでチケット（デジタル商品）を販売
  - 購入者にYouTube限定公開URLを配信
```

### F-08: ECサイト（Shopify 活用）

```
基盤: Shopify（自前EC構築しない → コスト大幅削減）

商品管理: Shopify管理画面で登録・在庫・注文を管理
  - 物販（アパレル、グッズ等）
  - イベントチケット（デジタル商品として登録）

アプリ側（Storefront API 経由）:
  - 商品一覧をアプリ内に表示
  - 商品詳細（画像・価格・在庫）
  - 購入 → Shopify Checkoutに遷移して決済
  - 注文履歴の表示

決済: Shopify Payments（クレカ・PayPay対応済み）

プラン別割引:
  - Shopify のディスカウントコード or 会員限定コレクションで対応

チケット販売フロー:
  1. イベントチケット = Shopifyのデジタル商品
  2. 購入完了 → メールでチケット（QRコード）送信
  3. イベント当日 → アプリの会員証 or チケットQRを提示
```

### F-09: タイムライン

```
形式: Instagram風のフィードUI

機能:
  - テキスト + 画像 + 動画の投稿
  - いいね・コメント
  - タブ分け（運営投稿 / コミュニティ等）

Instagram連携（要確認）:
  - 案A: Instagram投稿を自動取り込み（Graph API）
  - 案B: Instagram投稿の埋め込み表示
  - 案C: 独立したタイムライン（連携なし）
```

### F-10: 会員特典

```
想定特典:
  - グルメ系: 飲食店でドリンクサービス等
  - 不動産系: 仲介サービス連携（要確認）

利用方法:
  - アプリ内のデジタル会員証を店頭で提示
  - QRコード or 画面表示
  - 利用回数の管理

プラン別特典差:
  - 基本特典（全プラン共通）
  - 上位プラン限定特典（要確認）
```

### F-11: プッシュ通知

```
通知カテゴリ（ユーザーが個別にON/OFF設定）:
  - ライブ配信の開始
  - 新コンテンツ追加
  - EC新商品・イベント
  - 会員特典情報
  - 運営からのお知らせ

運営側:
  - セグメント配信（プラン別等）
  - スケジュール配信
```

### F-12: 管理画面

```
ダッシュボード:
  - 会員数（プラン別内訳）
  - 月次売上（サブスク + EC + 追加課金）
  - アクティブ率

管理機能:
  - 会員一覧・検索・プラン変更
  - コンテンツのアップロード・公開管理
  - ライブ配信の作成・アーカイブ管理
  - EC商品の登録・在庫・注文管理
  - 会員特典の追加・編集
  - プッシュ通知の作成・配信
  - 売上レポート・CSV出力
```
