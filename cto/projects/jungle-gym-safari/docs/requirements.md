# Jungle Gym Safari — 要件定義書

## 1. プロジェクト概要

サブスクリプション型オンラインコミュニティプラットフォーム。
美容・トレーニング・金融・健康・スタートアップなどの教材コンテンツを中心に、
ライブ配信・EC・会員特典を統合したオールインワンアプリ。

- **クライアント**: 株式会社JungleGYM
- **デザイン**: ロンハーマン / 西海岸テイスト
- **プラットフォーム**: Web（レスポンシブ） + モバイルアプリ（iOS / Android）
- **ローンチ**: 2026年8月完成 → 9月1日サービス開始

---

## 2. 機能要件

### 2.1 会員管理・認証

| 機能 | 詳細 |
|---|---|
| アカウント作成 | 名前、生年月日、メールアドレス、パスワード |
| ソーシャルログイン | Apple ID / Google / LINE（検討） |
| プロフィール管理 | アイコン、自己紹介、プラン情報表示 |
| 会員証機能 | アプリ内デジタル会員証（店舗提示用） |

### 2.2 サブスクリプション・課金

#### 3プラン構成

| プラン | 価格帯 | 内容 |
|---|---|---|
| **Bronze** | 低価格帯 | コンテンツ1つ無料選択 + 追加は課金 |
| **Silver** | 中価格帯 | コンテンツ3つ無料選択 + 追加は課金 |
| **Gold** | 高価格帯 | 全コンテンツ見放題 |

※ プラン名・価格はクライアントと要調整

#### 課金システム

- サブスクリプション（月額自動引き落とし）
- 個別コンテンツ課金（プラン外の追加購入）
- EC商品購入（単発決済）
- **決済手段**: クレジットカード / PayPay / 口座引き落とし

## 3. 非機能要件

| 項目 | 要件 |
|---|---|
| パフォーマンス | ページロード 2秒以内、動画再生開始 3秒以内 |
| スケーラビリティ | 初期1,000人 → 10,000人規模まで対応可能な設計 |
| セキュリティ | 決済情報のPCI DSS準拠、個人情報の暗号化 |
| 可用性 | 99.9% SLA（ライブ配信時は特に重要） |
| 対応端末 | iOS 16+、Android 12+、Chrome/Safari/Edge最新2バージョン |
| 多言語 | 初期は日本語のみ（将来的に英語対応検討） |

## 4. 外部サービス連携

### コスト最適化方針: 自前構築を最小限にし、既存プラットフォームを最大活用

| サービス | 用途 | 備考 |
|---|---|---|
| YouTube Live | ライブ配信基盤 | 無料。配信・チャット・アーカイブ全て込み |
| Shopify | EC（物販・チケット） | 商品管理・在庫・注文・決済が全て込み |
| Shopify Payments | EC決済（クレカ・PayPay） | Shopify内で完結。自前決済連携不要 |
| Stripe Billing | サブスク月額課金 | サブスク管理の定番 |
| Stripe Checkout | コンテンツ追加課金 | 都度課金 |
| CloudFront + S3 | 動画配信・CDN | コンテンツ配信 |
| Firebase (FCM) | プッシュ通知 | モバイル通知。無料 |
| Instagram Graph API | Instagram連携 | 投稿取り込み |
| SendGrid | メール配信 | ウェイティングリスト等 |
| Supabase | DB・認証 | PostgreSQL + Auth 一体型 |
