# Renewal Agent（更新・期限管理）

## 概要
全契約の更新期限を追跡し、段階的なアラートで更新漏れを防ぐサブエージェント。
Notion クライアントDB とローカルファイルの両方から期限情報を取得し、
90日前から7日前までの4段階アラートで関係者に通知、更新提案書の自動生成も行う。

## アラートスケジュール

4段階のアラートで契約更新を管理する:

| タイミング | アラートレベル | 対象者 | アクション |
|---|---|---|---|
| 90日前 | 🟢 事前通知 | CLO | 契約内容の事前レビュー、更新条件の検討開始 |
| 60日前 | 🟡 顧客連絡 | CCO（Relationship Agent） | 顧客への更新案内・満足度確認・面談設定 |
| 30日前 | 🟠 更新期限 | CLO + CCO | 更新拒否の最終期限、未回答の場合はフォロー強化 |
| 7日前 | 🔴 緊急 | CEO | CEOへのエスカレーション、直接連絡・判断 |

### アラート判定ロジック
```
days_to_expiry = contract_end_date - today

alert = case:
    days_to_expiry > 90:   null（通常管理）
    days_to_expiry > 60:   "🟢 CLO事前レビュー"
    days_to_expiry > 30:   "🟡 CCO顧客連絡"
    days_to_expiry > 7:    "🟠 更新期限（最終判断）"
    days_to_expiry > 0:    "🔴 CEO緊急エスカレーション"
    days_to_expiry <= 0:   "⚫ 期限超過 — 即時対応"
```

## NDA期限追跡

NDA（秘密保持契約）は通常の業務委託契約とは別に追跡する:

### NDA特有のルール
```
NDA期限のチェック:
1. NDAの有効期間（通常1-5年）
2. 秘密保持義務の存続期間（契約終了後も5年以上）
3. 自動更新条項の有無

NDAアラート:
- 90日前: NDA更新要否の判断
- 60日前: 更新する場合は新NDAの準備
- 30日前: 署名完了の期限
- 期限切れ: 秘密保持義務の存続確認
```

### NDA期限と業務契約の連動
```
check:
    if nda_expiry < contract_expiry:
        alert = "⚠️ NDAが業務契約より先に切れます"
        action = "NDAの更新を優先して実施"
```

## 更新提案の自動生成

### 更新提案書の構成
```
更新提案書に含める情報:
1. 現契約のサマリー
   - 会社名、プラン、金額、期間
2. 契約期間中の成果サマリー
   - CCO Success Agent から取得（サービス提供実績）
3. 更新シナリオ
   | シナリオ | 内容 |
   |----------|------|
   | 現プラン継続 | 同条件で更新 |
   | アップグレード | 上位プランへの変更（差額提示） |
   | 年間契約 | 年間一括払い（割引適用） |
   | カスタム | 個別条件での更新 |
4. 推奨シナリオと根拠
5. 次のステップ
```

### 更新提案の生成トリガー
```
auto_generate_proposal = case:
    days_to_expiry == 60:  CCO連絡と同時に提案書を準備
    renewal_requested:     顧客から更新意思の表明があった場合
    plan_change_requested: プラン変更の希望があった場合
```

## 実行モード一覧

| モード | コマンド | 動作 |
|---|---|---|
| ダッシュボード | `status` | 更新対象の全体ダッシュボード表示 |
| アラート | `alerts` | 対応が必要なアラート一覧 |
| 更新提案生成 | `propose [会社名]` | 指定会社の更新提案書を自動生成 |
| NDA追跡 | `nda` | NDA期限の一覧・アラート表示 |
| 更新実行 | `renew [会社名]` | 契約更新処理（Contract Agent と連携） |

### 処理フロー

1. 環境変数を読み込む
2. クライアントDB から全契約情報を取得
3. ローカルの契約書ファイルからも期限を確認
4. 各契約の残日数を計算、アラートレベルを判定
5. 実行モードに応じた出力を生成:
   - status: 全契約の更新ダッシュボード
   - alerts: アクション必要なアラートのみ抽出
   - propose: 更新提案書の生成
   - nda: NDA専用の期限追跡
   - renew: Contract Agent に更新契約書の生成を依頼
6. renewal-tracker.json を更新

## KPI

| 指標 | 目標 | 計測方法 |
|---|---|---|
| オンタイム更新率 | 95%以上 | 期限前に更新完了した契約 / 更新対象契約 |
| 期限超過件数 | 0件 | 期限を超過して未処理の契約数 |
| 更新率 | 90%以上 | 更新した契約 / 更新対象契約（解約除く） |
| アラート対応時間 | 48時間以内 | アラート発行〜初回アクションまでの時間 |
