# 領収書登録スキル

## 概要
Gmail、画像/PDF、テキスト入力から領収書情報を解析し、Notion領収書DBに登録する。

## 環境変数の読み込み
```bash
source ~/Desktop/クロードコード/.env
```

## 入力パターンの判定

### パターン0: 引数なし（`/receipt-add` のみ）
→ Gmail連携モード（下記「Gmail連携フロー」へ）

### パターン1: 画像/PDFパスが指定された場合
1. Readツールでファイルを読み込む
2. AIで日付・金額・支払先・明細を解析
3. 「登録内容の確認」へ進む

### パターン2: テキストが指定された場合
例: `/receipt-add Amazon 3980円 3/15 ソフトウェア`
1. テキストから日付・金額・支払先を解析
2. 「登録内容の確認」へ進む

---

## Gmail連携フロー

### ステップ1: 領収書メールの検索
Gmail MCP toolを使って領収書メールを検索する。

**検索クエリ:**
```
subject:(領収書 OR receipt OR 領収 OR 購入確認 OR ご利用明細) -label:領収書処理済
```

デフォルトは直近30日。ユーザーが期間を指定した場合はそれに従う。
例: `after:2026/01/01 before:2026/03/31`

`gmail_search_messages` ツールを使用する。

### ステップ2: 検索結果の一覧表示
結果を以下のフォーマットで表示:

```
━━━ Gmail領収書メール（直近30日）━━━
No. | 日付       | 送信元              | 件名                    | 状態
----|-----------|---------------------|------------------------|------
 1  | 2026-02-24 | AIゼミ              | 領収書 #2577-7170       | 未処理
 2  | 2026-02-21 | お名前.com          | 領収明細 2026年2月度     | 未処理
 3  | 2026-01-11 | Studio Inc.         | 領収書 #2184-2850       | 未処理
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

処理する番号を選択してください（複数: 1,2,3 / 全て: all）
```

**除外すべきメール:**
- 広告・プロモーション系（freeeの案内メール等、CATEGORY_PROMOTIONSラベル付き）
- 「領収書ダウンロード機能のお知らせ」等の通知系
- 実際の領収書・支払い確認ではないメール

判定基準: メール本文に具体的な金額（￥xxx, xxx円）が含まれているかどうか

### ステップ3: メール本文の読み取りと解析
ユーザーが選択したメールを `gmail_read_message` で読み取り、以下を抽出:

**抽出項目:**
- **日付**: メール本文中の支払日、なければメール送信日
- **金額**: ￥マーク or 円の前の数値（税込金額を優先）
- **支払先**: 送信元の会社名・サービス名
- **明細**: 商品名・サービス名
- **領収書番号**: あれば（Stripe系は領収書番号あり）
- **税区分**: 10%対象 or 8%対象 or 不明

**送信元別の解析パターン:**

#### Stripe系（`*@stripe.com`）
- 支払先: Fromヘッダーの表示名（例: "AIゼミ", "Studio Inc."）
- 金額: `￥XX,XXX` or `お支払い金額` の後の数値
- 日付: `お支払い日` の後の日付
- 明細: 明細行のテキスト
- 領収書番号: Subject or 本文中の `領収書番号 XXXX-XXXX`

#### お名前.com系（`server@onamae-support.jp`）
- 支払先: お名前.com
- 金額: `お支払い金額.....: XXX円` のパターン
- 日付: Subjectの年月度 or メール日付
- 明細: レンタルサーバー利用料
- カテゴリ: サーバー・ホスティング

#### 添付PDF系
- メール本文に金額情報がない場合、添付ファイルの存在を通知
- 「添付PDFがあります。ダウンロードして `/receipt-add <パス>` で登録してください」と案内

#### その他
- メール本文からAIで最善の解析を試みる

## Notion登録

### 領収書DBへの登録
```bash
curl -s -X POST "https://api.notion.com/v1/pages" \
  -H "Authorization: Bearer $NOTION_API_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{
    "parent": {"database_id": "$NOTION_RECEIPT_DB"},
    "properties": {
      "支払先": {"title": [{"text": {"content": "【支払先】"}}]},
      "日付": {"date": {"start": "【YYYY-MM-DD】"}},
      "金額": {"number": 【金額（数値）】},
      "カテゴリ": {"select": {"name": "【カテゴリ】"}},
      "税区分": {"select": {"name": "【税区分】"}},
      "ステータス": {"select": {"name": "未確認"}},
      "ソース": {"select": {"name": "【Gmail/手動/写真】"}},
      "明細": {"rich_text": [{"text": {"content": "【明細】"}}]},
      "領収書番号": {"rich_text": [{"text": {"content": "【領収書番号】"}}]},
      "メモ": {"rich_text": [{"text": {"content": "【メモ】"}}]}
    }
  }'
```

### 登録完了後の表示
```
✓ 登録完了: AIゼミ ¥12,000（2026-02-24）
✓ 登録完了: お名前.com ¥484（2026-02-21）

合計 2件 / ¥12,484 を登録しました。
```

---

## 注意事項
- NOTION_RECEIPT_DB が .env に存在しない場合は、「領収書DBがまだ作成されていません。`/notion 領収書DBを作成して` で先にDBを作成してください。」と案内する
- 同じ領収書番号が既にNotionに登録されている場合は重複を警告する
- 金額は必ず税込の数値（カンマなし）で登録する
