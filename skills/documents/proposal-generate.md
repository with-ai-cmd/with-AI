# 提案書自動生成スキル

## 概要
クライアント名と課題感・要望を入力すると、NotionクライアントDB/コンタクトDBから情報を取得し、AIKOMONのサービス知識に基づいた提案書を自動生成する。

## 環境変数の読み込み
以下のファイルから環境変数を読み込んでください:
- `~/Desktop/クロードコード/.env`

## 入力の判定

$ARGUMENTSの解析:
- **パターン1**: 会社名のみ → クライアント/コンタクト情報を取得後、課題感をユーザーに確認
- **パターン2**: 会社名 + 課題感/要望 → そのまま提案書生成に進む
- **パターン3**: 引数なし → 会社名を確認してから進行

## 手順

### 1. クライアント/コンタクト情報の取得

まずクライアントDBを検索（既存顧客かどうか確認）:
```bash
curl -s -X POST "https://api.notion.com/v1/databases/$NOTION_CLIENT_DB/query" \
  -H "Authorization: Bearer $NOTION_API_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{
    "filter": {
      "property": "会社名",
      "title": {"contains": "【検索キーワード】"}
    }
  }'
```

見つからなければコンタクトDBを検索:
```bash
curl -s -X POST "https://api.notion.com/v1/databases/$NOTION_CONTACT_DB/query" \
  -H "Authorization: Bearer $NOTION_API_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{
    "filter": {
      "or": [
        {"property": "氏名", "title": {"contains": "【検索キーワード】"}},
        {"property": "会社名", "rich_text": {"contains": "【検索キーワード】"}}
      ]
    }
  }'
```

### 2. ミーティング履歴の取得（あれば）

クライアントまたはコンタクトのページIDでミーティングDBを検索し、過去の打ち合わせ内容・課題感を把握する:
```bash
curl -s -X POST "https://api.notion.com/v1/databases/$NOTION_MEETING_DB/query" \
  -H "Authorization: Bearer $NOTION_API_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{
    "filter": {
      "property": "クライアント",
      "relation": {"contains": "【ページID】"}
    },
    "sorts": [{"property": "日時", "direction": "descending"}]
  }'
```

### 3. 情報の整理と確認

取得した情報を以下の形で表示し、ユーザーに確認する:

```
━━━ 提案書作成情報 ━━━
会社名:     【会社名】
担当者:     【担当者名】
契約状況:   【既存クライアント（プラン名）/ 新規見込み】
直近MTG:    【最新ミーティングの要約（あれば）】
課題感:     【ユーザーが入力した課題 or MTGから抽出した課題】
━━━━━━━━━━━━━━━━━━━━━
```

課題感が不明な場合はユーザーに確認する。

### 4. 提案書の生成

以下のAIKOMONサービス知識を踏まえ、クライアントの課題に最適な提案書を生成する。

#### 参照すべきサービス知識
必ず `~/Desktop/クロードコード/company/services.md` を読み込んでサービス詳細を把握すること。

#### AIKOMONのコアメッセージ（提案に必ず反映する）
- AIを「ツール」として使うだけでは業務効率・意思決定スピードは変わらない
- 「AIを"使う"会社」から「AIと"働く"会社」へ変革する
- 社内に「技術とビジネスをつなぐAI責任者」が不在であることが根本原因
- AIKOMONはアドバイス屋ではなく、業務設計→AI実装→自動化開発→定着支援まで一気通貫で責任を持つ
- 全社員を「AI使い」へ引き上げる教育とサポート
- 設計して終わりではなく、成果が出るまで伴走する実行責任型サービス

#### 提案書の構成

```markdown
# 【会社名】様 ご提案書

## 1. 貴社の現状と課題
（ヒアリング内容・ミーティング記録から整理）

## 2. 課題の本質
（なぜその課題が発生しているのか、AIKOMONの視点で分析）

## 3. ご提案内容
（課題に対する具体的な解決策をAIKOMONのサービスで提案）

### 3-1. 実施内容
（具体的に何をやるか、フェーズに分けて記載）

### 3-2. 期待される効果
（定量・定性両面で記載）

### 3-3. 実施スケジュール（目安）
（フェーズごとのタイムライン）

## 4. ご提案プラン・費用
（クライアントの規模・課題に合わせた最適プランを提案）
- AI相談プラン: 49,800円/月 — 月1回相談でAI活用の方向性を明確に
- AIKOMONプラン: 25万円/月【推奨】— 従業員25名程度まで、実働型AI責任者
- プレミアムプラン: 50万円/月〜 — 従業員30名以上、複数部署横断支援

## 5. なぜAIKOMONか
- 提案から仕組み化・定着まで一気通貫で伴走
- 研修会社でも開発会社でもない「間を埋める」存在
- 経営者の思考・判断基準をAIに落とし込める独自手法

## 6. ネクストステップ
（次に何をすればよいかを明確に提示）
```

#### 提案書作成のルール
- クライアントの業種・規模・課題に合わせてカスタマイズする
- 既存クライアントの場合は現在の契約プランを踏まえ、追加提案やアップグレード提案を行う
- 抽象的な表現は避け、具体的なアクションと成果を記載する
- 費用は全プラン記載しつつ、推奨プランにはその理由を明記する
- 「4. ご提案プラン・費用」では、クライアントに不要なプランは省略してもよい

### 5. 出力

提案書をMarkdownファイルとして保存:
```
~/Desktop/クロードコード/proposals/【会社名】_提案書_【日付YYYYMMDD】.md
```

proposalsディレクトリが存在しない場合は作成する。

### 6. PPTX化の確認
保存後、ユーザーに以下を確認する:
- 「提案書をPowerPointにしますか？」 → Yesなら /pptx で生成を案内
- 「ミーティングを記録しますか？」 → Yesなら /meeting-add を案内

### 7. NotionクライアントDBの更新
クライアントDBに登録済みの場合、最終連絡日を今日の日付に更新する。
