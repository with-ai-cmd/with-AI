# 見積書自動生成スキル

## 概要
会社名・サービス・期間を入力すると、NotionクライアントDB/コンタクトDBから情報を取得し、with-AI株式会社のサービス料金に基づいた見積書をMarkdownで自動生成する。

## 環境変数の読み込み
以下のファイルから環境変数を読み込んでください:
- `~/Desktop/with-AI/skills/documents/クロードコード/.env`

## 入力の判定

$ARGUMENTSの解析:
- **パターン1**: 会社名のみ → クライアント/コンタクト情報を取得後、サービス内容・期間をユーザーに確認
- **パターン2**: 会社名 + サービス名 → 規模・期間を推定または確認して見積書生成に進む
- **パターン3**: 会社名 + サービス名 + 期間 → そのまま見積書生成に進む
- **パターン4**: 引数なし → 会社名を確認してから進行

## 手順

### 1. クライアント/コンタクト情報の取得

まずクライアントDBを検索（既存顧客かどうか確認）:
```bash
curl -s -X POST "https://api.notion.com/v1/databases/$NOTION_CLIENT_DB/query" \
  -H "Authorization: Bearer $NOTION_API_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{
    "filter": {
      "property": "会社名",
      "title": {"contains": "【検索キーワード】"}
    }
  }'
```

見つからなければコンタクトDBを検索:
```bash
curl -s -X POST "https://api.notion.com/v1/databases/$NOTION_CONTACT_DB/query" \
  -H "Authorization: Bearer $NOTION_API_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{
    "filter": {
      "or": [
        {"property": "氏名", "title": {"contains": "【検索キーワード】"}},
        {"property": "会社名", "rich_text": {"contains": "【検索キーワード】"}}
      ]
    }
  }'
```

Notionに情報がない場合はユーザーに以下を確認する:
- 会社名（正式名称）
- 担当者名
- 住所（任意）

### 2. サービス・規模・期間の確認

以下の情報が不足していればユーザーに確認する:

- **対象サービス**: どのプランか（下記料金表を参照）
- **会社規模**: 従業員数（プラン判定に使用）
- **契約期間**: 何ヶ月契約か（デフォルト: 6ヶ月）
- **初期費用の有無**: セットアップが必要か

### 3. 情報の整理と確認

取得した情報を以下の形で表示し、ユーザーに確認する:

```
━━━ 見積書作成情報 ━━━
会社名:       【会社名】
担当者:       【担当者名】
対象サービス: 【サービス名】
会社規模:     【従業員数】名
契約期間:     【X】ヶ月
初期費用:     【あり/なし】
━━━━━━━━━━━━━━━━━━━━━
```

内容に問題がなければ見積書生成に進む。

### 4. 見積書の生成

#### 料金表（税別）

| プラン | 月額（税別） | 対象規模 | 備考 |
|---|---|---|---|
| AIコンサルテーション（AI相談プラン） | ¥49,800 | - | 月1回の相談でAI活用の方向性を明確に |
| AIKOMONプラン | ¥250,000 | 従業員25名程度まで | 実働型AI責任者として伴走 |
| プレミアムプラン | ¥500,000〜 | 従業員30名以上 | 複数部署横断支援、カスタマイズ対応 |
| AI研修・セミナー | ¥100,000〜300,000/回 | - | 単発またはシリーズ開催 |
| システム開発 | 別途見積 | - | 要件に応じて個別見積 |

#### 見積書テンプレート

以下の構成でMarkdownの見積書を生成する:

```markdown
# 御見積書

---

**見積番号**: EST-【YYYYMMDD】-001
**発行日**: 【YYYY年MM月DD日】
**有効期限**: 【発行日から30日後の日付】

---

## 発行元

**with-AI株式会社**
代表取締役 鹿内 海斗
メール: info@with-ai.co.jp
URL: https://with-ai.co.jp

---

## 宛先

**【会社名】** 御中
【担当者名】 様

---

## 件名

【サービス名】に関する御見積

---

## 御見積金額

### **合計金額: ¥【合計金額】（税別）**
### **合計金額: ¥【合計金額×1.1】（税込）**

---

## 明細

| No. | 項目 | 数量 | 単位 | 単価（税別） | 金額（税別） |
|-----|------|------|------|-------------|-------------|
| 1 | 【サービス名】 月額利用料 | 【契約月数】 | ヶ月 | ¥【月額】 | ¥【月額×月数】 |
| 2 | 初期セットアップ費用 | 1 | 式 | ¥【初期費用】 | ¥【初期費用】 |
|  | **小計** |  |  |  | **¥【小計】** |
|  | **消費税（10%）** |  |  |  | **¥【税額】** |
|  | **合計（税込）** |  |  |  | **¥【税込合計】** |

※ 初期セットアップ費用が不要の場合はNo.2の行を削除する。

---

## サービス内容

【選択されたプランに応じたサービス内容を記載】

### AIコンサルテーション（AI相談プラン）の場合:
- 月1回のオンライン相談（60分）
- AI活用の方向性アドバイス
- チャットでの質問対応（月5件まで）

### AIKOMONプランの場合:
- 社外AI責任者として業務設計からAI実装まで伴走
- 全社員向けAI活用サポート（チャット無制限）
- 月2回の定例ミーティング
- 業務プロセスのAI化提案・実装
- 社内AIツールの選定・導入支援

### プレミアムプランの場合:
- AIKOMONプランの全内容を含む
- 複数部署横断でのAI導入支援
- 週1回の定例ミーティング
- カスタムAIツール・ワークフロー開発
- 経営層向けAI戦略レポート（月次）

### AI研修・セミナーの場合:
- 研修テーマ: 【テーマ】
- 対象人数: 【人数】名
- 所要時間: 【時間】
- 形式: オンライン / オフライン

---

## お支払い条件

- **支払方法**: 銀行振込
- **支払サイト**: 月末締め翌月末払い
- **初期費用**: 契約締結後14日以内にお支払い
- **月額費用**: サービス開始月より毎月発生

---

## 備考

- 本見積書の有効期限は発行日より30日間です。
- 金額は全て税別表記です（消費税10%が別途加算されます）。
- 契約期間中の途中解約は、1ヶ月前までに書面にてご連絡ください。
- 上記金額にはお客様環境固有のカスタマイズ開発費用は含まれておりません。
```

#### 見積書作成のルール
- 金額はカンマ区切りで表示する（例: ¥1,500,000）
- 消費税は10%で計算する
- 見積番号は `EST-YYYYMMDD-001` 形式（同日複数の場合は連番）
- プレミアムプランの場合、月額は顧客の規模に応じて ¥500,000 を基準に調整する
- AI研修・セミナーの場合、内容・時間に応じて ¥100,000〜300,000 の範囲で設定する
- システム開発の場合、要件をヒアリングして工数ベースで見積もる
- 既存クライアントへの追加サービスの場合、既存契約との整合性に注意する

### 5. 出力

見積書をMarkdownファイルとして保存:
```
~/Desktop/with-AI/company/proposals/【会社名】_見積書_【YYYYMMDD】.md
```

proposalsディレクトリが存在しない場合は作成する。

保存後、見積書の内容を画面に表示する。

### 6. 追加アクションの確認

保存後、ユーザーに以下を確認する:
- 「見積書をPDFに変換しますか？」 → Yesならpandoc等でPDF生成を実行
- 「見積書をメールで送付しますか？」 → YesならGmailドラフトを作成（宛先・件名・本文を自動生成し、見積書を添付）
- 「提案書も作成しますか？」 → Yesなら /proposal-generate を案内

### 7. NotionクライアントDBの更新

クライアントDBに登録済みの場合、最終連絡日を今日の日付に更新する:
```bash
curl -s -X PATCH "https://api.notion.com/v1/pages/【ページID】" \
  -H "Authorization: Bearer $NOTION_API_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{
    "properties": {
      "最終連絡日": {
        "date": {"start": "【今日の日付 YYYY-MM-DD】"}
      }
    }
  }'
```
