# 契約管理スキル

## 概要
クライアントの契約ライフサイクルを管理する。
契約書の格納場所管理、契約期限チェック、更新通知、ステータス管理を行う。

## 環境変数の読み込み
- `{{WITHAI_ROOT}}/skills/documents/クロードコード/.env`

## データソース
- **Notion人脈DB**: `326d725a-9f17-81cd-82a9-d0c9fd8873eb`（大元のコンタクト情報）
- **NotionクライアントDB**: `611e82b2-a57e-4471-b62d-28dcb1d8ffa4`（契約情報）
- **ローカル契約書**: `{{WITHAI_ROOT}}/clients/{会社名}/contracts/`
- **契約書テンプレート**: `{{WITHAI_ROOT}}/company/contracts/source/`

---

## コマンド分岐

### パターン1: 引数なし or 「契約一覧」
→ 契約中クライアントの一覧表示 + アラートチェック

### パターン2: 「契約確認 {会社名}」
→ 特定クライアントの契約詳細表示

### パターン3: 「契約登録 {会社名}」
→ 新規契約の登録フロー

### パターン4: 「契約更新 {会社名}」
→ 契約更新処理

### パターン5: 「契約書格納 {会社名}」
→ 契約書ファイルのローカル格納

### ステップ3: アラート判定ロジック
```python
today = date.today()

for client in clients:
    # 契約終了間近（60日以内）
    if 契約終了日 and (契約終了日 - today).days <= 60:
        alert("契約終了間近", days_remaining)

    # 契約書ファイル未格納
    contract_dir = f"{{WITHAI_ROOT}}/clients/{会社名}/contracts/"
    if not os.listdir(contract_dir):
        alert("契約書未格納")

    # 契約開始日が過ぎているのに契約状況が「契約準備中」
    if 契約状況 == "契約準備中" and 契約開始日 < today:
        alert("契約開始日超過・ステータス未更新")
```

## パターン3: 新規契約登録

### ステップ1: 人脈DBから情報取得
指定された会社名で人脈DBを検索し、コンタクト情報を取得。

### ステップ2: 契約情報の入力
以下を確認:
- 契約プラン（選択肢を提示）
- 月額（プランのデフォルト値を提示）
- 契約開始日
- 契約終了日（空欄 = 自動更新）
- 住所
- 代表者名
- 請求先メール
- 請求サイクル（デフォルト: 月末締め翌月末払い）

### ステップ3: クライアントDBに登録
Notion MCP toolでクライアントDBにページを作成。

### ステップ4: 人脈DBのステータスを「成約」に更新
Notion MCP toolで該当コンタクトのステータスを更新。

### ステップ5: ローカルディレクトリ作成
```bash
mkdir -p {{WITHAI_ROOT}}/clients/{会社名}/{contracts,invoices,notes}
```

### ステップ6: 次のアクション提案
```
✅ 契約登録完了: 株式会社AAA

次のアクション:
  1. 契約書を生成しますか？ → /contract-generate AAA
  2. 初月の請求書を生成しますか？ → /invoice-generate AAA
```

## パターン5: 契約書格納

### ステップ1: 会社名でクライアントを特定
### ステップ2: 格納するファイルのパスを確認
ユーザーがファイルパスを指定、またはデスクトップ等から自動検索:
```bash
find ~/Desktop -name "*契約書*" -newer {{WITHAI_ROOT}}/clients/{会社名}/contracts/ -maxdepth 2 2>/dev/null
```

### ステップ3: ファイルをクライアントディレクトリにコピー
```bash
cp "{元のパス}" {{WITHAI_ROOT}}/clients/{会社名}/contracts/{契約種類}_{締結日}.pdf
```
