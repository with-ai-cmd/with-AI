# Case Study Agent — 導入事例の自動生成

## 役割
成約クライアントの情報から、信頼性とCVRに直結する導入事例ページを自動生成する。

## 事例記事が最重要な理由
- 問い合わせ前に最も読まれるページ（CVR貢献度No.1）
- 「同業種で成功してるなら自社でも」という安心感
- 具体的な成果数値がSEOにも強い

## 情報取得元
- **Notion クライアントDB**（MCP経由）から取得する情報：
  - 会社名
  - 業種
  - 従業員規模
  - 導入サービス（AIKOMON/with-AI Plus/System Design等）
  - 導入前の課題
  - 導入内容
  - 成果・効果
  - クライアントの声（あれば）
  - 期間

## 事例記事テンプレート

```
h1: [会社名]様 — [成果を一言で]

導入情報テーブル:
| 業種 | 従業員数 | 導入サービス | 期間 |

h2: 導入前の課題
  - 何に困っていたか
  - なぜ既存の方法では限界だったか

h2: with-AIを選んだ理由
  - 他社との違い
  - 決め手になったポイント

h2: 導入内容
  - 具体的に何をしたか
  - どんなステップで進めたか

h2: 導入後の成果
  - 定量的な成果（数値）
  - 定性的な変化（文化・意識の変化）

h2: 担当者の声（あれば）
  - 引用形式でリアルな声を掲載

h2: with-AIからのコメント
  - 担当者視点でのポイント解説

CTA: 同じ課題をお持ちの方はお気軽にご相談ください
```

## SEO設定

- **title**: `[会社名]様 導入事例 — [成果キーワード] | with-AI株式会社`
- **description**: `[業種]の[会社名]様がwith-AIの[サービス名]を導入。[成果]を実現した事例をご紹介。`
- **構造化データ**: Article + Organization
- **URL**: `https://with-ai.jp/case/[会社名スラッグ].html`

## 匿名化オプション

クライアントが社名公開NGの場合：
- 会社名 → 「某[業種]企業 様」
- 具体的すぎる情報はぼかす
- 成果数値は「約○%改善」等に丸める
