# HP改修チーム（CMO × CTO 合同）

## トリガー
「HP改修」「HP改善」「ホームページ修正」「サイト改修」「CRO改善」「LP改善」

## 概要
with-ai.jp のUI/UX改善・ページ追加・導線最適化を行う合同チーム。
**何を改修するか = CMO管轄（データドリブン）、どう改修するか = CTO管轄（実装・デプロイ）。**

## 組織図

```
CMO（改修企画・効果検証）               CTO（設計・実装・デプロイ）
  │                                     │
  ├── analytics    ──データ分析──→       ├── HTML/CSS/JS実装
  ├── seo-optimizer ──SEO要件──→        ├── レスポンシブ対応
  ├── cro改善      ──改善案──→          ├── デザインシステム適用
  └── report       ──効果検証──→        └── Xserverデプロイ
```

## 連携フロー

```
Step 1: CMO側 — 課題発見（データ分析）
  GA4 / Search Console / ヒートマップ から:
  - 離脱率が高いページの特定
  - CVR（問い合わせ率）が低いページ
  - 直帰率が高いランディングページ
  - ユーザーフローのボトルネック
  - モバイル vs PC の行動差異

Step 2: CMO側 — 改修企画書
  以下のフォーマットでCTOに依頼:
  ┌─────────────────────────────────┐
  │ 改修依頼書                       │
  ├─────────────────────────────────┤
  │ 対象ページ: /aikomon.html       │
  │ 課題: CTAまでの離脱率 80%       │
  │ 仮説: ファーストビューにCTAがない │
  │ 改修内容:                        │
  │   - ヒーロー下にCTAボタン追加    │
  │   - 料金表セクション追加         │
  │   - お客様の声セクション追加     │
  │ SEO要件: h2変更なし、内部リンク追加│
  │ 優先度: P1                       │
  │ 期待効果: CVR 0.3% → 0.8%      │
  └─────────────────────────────────┘

Step 3: CTO側 — 技術設計
  - 既存デザインシステムとの整合性確認
  - モバイル/PC両対応のレイアウト設計
  - 既存ページへの影響範囲確認
  - 実装方針を決定

Step 4: CTO側 — 実装・デプロイ
  - HTML/CSS/JS の修正
  - デザインシステム準拠（Albert Sans / Zen Kaku Gothic New / ガラスカード）
  - レスポンシブ確認
  - デプロイ:
    rsync -avz -e "ssh" ~/Desktop/hp/with-AI\ HP/ xserver:~/with-ai.jp/public_html/

Step 5: CMO側 — 効果検証（デプロイ1週間後）
  - GA4 で改修前後の比較
  - CVR / 離脱率 / 滞在時間の変化
  - 効果が出ていなければ再改修案を検討
```

## 改修タイプ別フロー

### A. CRO改善（コンバージョン最適化）
```
CMO: 離脱分析 → CTA改善案
CTO: ボタン配置・デザイン変更 → デプロイ
CMO: A/Bテスト or 前後比較で効果測定
```

### B. 新規ページ追加
```
CMO: ページ企画（ターゲット・キーワード・構成案）
CTO: HTML作成 → デザインシステム適用 → ナビ更新 → サイトマップ更新 → デプロイ
CMO: Search Consoleでインデックス確認 → 流入追跡
```

### C. 既存ページリニューアル
```
CMO: 現状データ分析 → 改善ポイント特定 → ワイヤーフレーム
CTO: 実装 → デプロイ
CMO: 効果検証
```

### D. URL変更・ページ統廃合
```
CMO: 統廃合の判断（重複コンテンツ、低パフォーマンスページ）
CTO: 301リダイレクト設定 → サイトマップ更新 → デプロイ
CMO: Search Consoleでリダイレクト確認 → インデックス状況追跡
```

## デザインシステム（CTO準拠）
- フォント: Albert Sans / Zen Kaku Gothic New / JetBrains Mono
- 背景: #EDEDF0 + グリッドパターン
- カード: ガラスモーフィズム（rgba(255,255,255,.45) + backdrop-filter）
- アニメーション: GSAP + ScrollTrigger
- カラー: `--blue`, `--cyan`, `--dark`（既存CSS変数）
- レスポンシブ: モバイルファースト + ハンバーガーメニュー
