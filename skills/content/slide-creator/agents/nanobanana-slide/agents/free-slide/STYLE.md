# free-slide スタイルルール

フリースタイル画像スライドの品質基準・共通ルールを定義する。
ホワイトボードスライドのように厳密な「4色マーカー縛り」はないが、プロ品質を担保するためのルールは存在する。

## 絶対禁止事項（全スライド共通）

以下をプロンプトに含めると、AIが画像内にテキストとして描画してしまうリスクがある：

- **フォント名**: Permanent Marker, Caveat, Kalam, Arial, Helvetica, Times, Noto Sans 等
- **ポイント数**: 14pt, 18pt, 24pt, 48pt, 72pt 等
- **カラーコード**: #000000, #FF0000, rgb(0,0,0), hsl(200,50%,50%) 等
- **ピクセル指定**: 1920x1080px, 200px from left 等
- **技術的比較**: "size equivalent to", "similar to [font]" 等
- **UIコンポーネント名**: button, dropdown, modal, tooltip 等

### 代わりに使用する表現

| 禁止 | 代替 |
|---|---|
| `#2563eb` | `bright blue` / `vivid blue` |
| `48pt` | `very large` |
| `Arial font` | `clean sans-serif style text` |
| `200px margin` | `generous spacing` |
| `1920x1080` | （指定不要、API設定で制御） |

## デザイン品質基準

### 1. 一貫性ルール

同一スライドセット内では以下を統一する：

- **カラーパレット**: 基調色2〜3色 + アクセント1〜2色を全スライドで共通
- **テキストスタイル**: タイトル・本文・注釈のサイズ感を全スライドで統一
- **レイアウトグリッド**: マージン・タイトル位置・コンテンツエリアを全スライドで揃える
- **イラストスタイル**: 同一セット内でイラストのタッチを統一（リアル↔フラット↔手描きを混ぜない）
- **トーン**: 全スライドで同じ世界観・雰囲気を維持

### 2. 視認性ルール

- テキスト量は最小限に。1スライド最大5ポイント
- テキストと背景のコントラストを十分確保
- 重要な情報を視覚的に強調（サイズ・色・位置で差をつける）
- 余白を十分に取る（詰め込みすぎない）
- 文字は必ず水平（斜め・カーブは禁止）

### 3. 構図ルール

- 情報の視覚的階層を明確に（タイトル > サブタイトル > 本文 > 注釈）
- 1スライド1メッセージの原則（情報を分散させない）
- ビジュアル要素（イラスト・図解・アイコン）を積極的に使用
- テキストだけのスライドは避ける

## カラーパレットの指針

ユーザーが色を指定しない場合、テーマに基づき以下から選択・アレンジする：

### プロフェッショナル系
- 基調: white, light gray
- メイン: dark navy, charcoal
- アクセント: bright blue, teal

### テクノロジー系
- 基調: dark background (deep navy or charcoal black)
- メイン: white, light gray
- アクセント: electric blue, neon green, bright purple

### エデュケーション系
- 基調: white, warm cream
- メイン: dark gray, warm brown
- アクセント: bright orange, sky blue, leaf green

### クリエイティブ系
- 基調: off-white or bold color
- メイン: contrasting dark or bright
- アクセント: coral, gold, vivid pink

### カジュアル・ポップ系
- 基調: white or pastel
- メイン: bright primary colors
- アクセント: contrasting vivid colors

**重要**: 色はすべて色名（英語）で記述する。カラーコードは絶対に使用しない。

## レイアウトバリエーション

フリースタイルでは以下のレイアウトを自由に組み合わせる：

| レイアウト | 説明 | 適した内容 |
|---|---|---|
| `title-hero` | 大きなタイトル + 背景ビジュアル | 表紙、セクション区切り |
| `split` | 左右2分割（テキスト + ビジュアル） | 概念説明、Before/After |
| `grid` | 2x2 or 3x3グリッド | 複数要素の並列紹介 |
| `full-visual` | 全面ビジュアル + オーバーレイテキスト | インパクト重視 |
| `data-viz` | グラフ・チャート中心 | 数値・比較データ |
| `timeline` | タイムライン・プロセスフロー | 歴史、ステップ説明 |
| `quote` | 引用・キーメッセージ中心 | 強調したい一文 |
| `icon-grid` | アイコン + ラベルの整列 | 機能紹介、サービス一覧 |
| `comparison` | 左右 or 上下の比較 | 2つの概念・選択肢 |
| `diagram` | 大きな図解 | 構造・関係性 |

**重要**: 連続するスライドで同じレイアウトを使わない。バリエーションを持たせる。

## テキストサイズの相対表現

| 用途 | サイズ表現 | 補足 |
|---|---|---|
| メインタイトル（表紙） | `very large, dominant` | スライドの主役 |
| ページタイトル | `large, prominent` | 明確に目立つ |
| サブタイトル | `medium-large` | タイトルより控えめ |
| 本文・箇条書き | `medium, readable` | 通常の読みやすいサイズ |
| 注釈・キャプション | `small` | 補足情報 |
| ページ番号・ラベル | `tiny` | 目立たない |

## プロンプト構成の原則

1. **シーン設定**: 全体の雰囲気・背景を最初に記述
2. **レイアウト**: 要素の配置を方向で記述（top, center, left side, bottom-right 等）
3. **テキスト内容**: 実際に表示するテキストを明示
4. **ビジュアル要素**: イラスト・図解の内容と配置を記述
5. **スタイル指示**: 色・雰囲気・タッチの統一指示

## white-board-slide との明確な差別化

| 観点 | free-slide | white-board-slide |
|---|---|---|
| 背景 | 自由（白、ダーク、グラデーション等） | 白（ホワイトボード面）固定 |
| テキスト | プリント風・モダン・手書き等自由 | 手書きマーカー風のみ |
| 色数 | 制限なし | 4色（黒青赤緑）のみ |
| イラスト | フラット、3D、リアル、手描き等自由 | ホワイトボードの落書き風のみ |
| フレーム | あり/なし自由 | なし固定 |
| 雰囲気 | テーマに応じて柔軟 | 教室のホワイトボード固定 |
