3章 プロンプト
プロンプトとは？
プロンプトとは、生成AIに入力する指示文のことです。
レストランの注文に例えると、メニューを見て「カルボナーラをください」と伝えるのがプロンプト。AIはシェフとして、その注文どおりの料理を作ります。
ひとことで言うと
プロンプト = AIへの「お願い」や「質問」をテキストで書いたもの

プロンプトの4つの構成要素
1. 指示（Instruction）
AIに何をしてほしいかを伝える部分
例：「要約してください」「翻訳して」
2. 文脈（Context）
背景情報や前提条件
例：「初心者向けに」「ビジネスメールとして」
3. 入力データ（Input）
処理してほしい対象のテキストやデータ
例：「以下の文章を〜」
4. 出力形式（Output Format）
どんな形で答えてほしいか
例：「箇条書きで」「表形式で」「JSON形式で」
この4要素を意識するだけで、プロンプトの質が格段に上がる

コツ③ 役割（ロール）を与える
「〜の専門家として答えて」と指定すると回答の深さが変わる。
役割の指定効果
小学校の先生としてやさしい言葉でかみ砕いた説明に
マーケティング専門家として市場データや戦略的な視点が加わる
プログラマーとしてコードの品質やバグの観点で評価
栄養士として栄養バランスや健康面を考慮した提案
Point
同じ質問でも、役割を変えるだけでまったく違う切り口の回答が得られる

コツ⑤ 条件と制約を設定する
やってほしいことだけでなく、やってほしくないことも明示する。
制約の種類具体例
文字数「300文字以内で」「500文字程度で」
対象レベル「専門用語を使わずに」「初心者向けに」
トーン「カジュアルな口調で」「ビジネス文書として」
除外条件「〜には触れないで」「英語は使わずに」
数量「3つに絞って」「最大5項目で」
条件をつける = 不要な情報を削り、必要な情報だけを残す

コツ⑦ 例文を示す（Few-Shot）
「こんな感じで書いて」とお手本を添えると形式を合わせてくれる。
Few-Shot Example 以下の形式で、他の商品の紹介文も書いてください。 【例】 商品名：有機グリーンティー 特徴：京都産の茶葉を使用。渋みが少なく、まろやかな味わい。 おすすめシーン：朝の目覚めの一杯に。 では、「ダークチョコレート」の紹介文をお願いします。
Point
お手本は1〜3個が効果的。多すぎるとプロンプトが長くなりすぎて逆効果。

プロンプト作成 7つのコツ
  • 具体的に書く ― 5W1Hを意識
  • 出力形式を指定 ― 箇条書き・表形式
  • 役割を与える ― 「〜の専門家として」
  • 背景情報を伝える ― 目的・対象者
  • 条件を設定 ― 文字数・トーン
  • 一つずつ指示 ― タスクを分割
  • 例文を示す ― Few-Shotの考え方

3章 プロンプト
よくある誤解
誤解 1
「長く書けば良い回答が返る」→ ポイントを絞った短い指示のほうが効果的なことが多い
誤解 2
「1回で完璧な回答が出るはず」→ 対話を繰り返して改善するのが正しい使い方
誤解 3
「万能のプロンプトが作れる」→ タスクや目的で最適なプロンプトは変わる。カスタマイズが大切

まとめ
  • プロンプトは、生成AIに入力する指示文のこと
  • 構成要素は 指示・文脈・入力データ・出力形式 の4つ
  • 具体的に書き、5W1Hを意識するだけで精度が大きく向上
  • 役割と背景情報を与えると回答の質がさらに上がる
  • 条件設定で不要な情報を削り、必要な情報だけを残す
  • 複雑なタスクは分割して段階的に指示する
  • 例文（Few-Shot）で出力の形式を揃えられる
  • 完璧を求めず、AIとの対話を繰り返して改善するのが上達の近道
