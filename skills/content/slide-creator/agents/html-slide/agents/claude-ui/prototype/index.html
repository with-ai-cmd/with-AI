「人間がAIを監視する」ってどういうこと？
これはHITL（Human-in-the-Loop）と呼ばれる考え方です。

簡単に言うと、AIの判断や出力を、人間が確認・修正してから使う運用のことです。

自動運転車で例えると、AIが運転を担当しますが、ドライバーがいつでもハンドルを握れる状態を保つのがHITLです。

なぜ重要かというと：
  • AIはハルシネーション（事実と異なる回答）を起こすことがある
  • AIの判断にバイアスが含まれることがある
  • 法的・倫理的な責任は人間が負う必要がある
  • 業界によっては法規制で人間の監視が義務化されている

監視のレベルにはどんな種類がある？
監視レベルは3段階に分けられます：
AI監視の3レベル
Level 1: Human-in-the-Loop（全件確認） → AIの出力を人間が毎回チェック。医療・法律向き Level 2: Human-on-the-Loop（抜き打ち確認） → AIが自動処理し、人間が定期的にサンプル監視 Level 3: Human-out-of-the-Loop（例外時のみ） → ほぼ自動化。異常検知時だけ人間が介入
リスクが高い業務ほどLevel 1に近づけ、定型的な業務はLevel 3で効率化するのが基本です。

社内ガイドラインには何を書けばいい？
社内AIガイドラインの必須項目は6つあります：
  1. 利用目的の明確化 — どの業務でAIを使ってよいか
  2. 入力禁止データの定義 — 個人情報・機密情報・顧客データ等
  3. 出力の確認義務 — ファクトチェックの手順とルール
  4. 利用可能ツールの指定 — 承認済みAIサービスのリスト
  5. 責任の所在 — AI出力の最終責任は誰にあるか
  6. インシデント対応 — 問題発生時の報告・対処フロー
まずはシンプルに始めて、運用しながら改善していくのがおすすめです。最初から完璧を目指すと、いつまでも策定が終わりません。

  • 国や地域の法規制が追いついていない部分がある

HITL

HOTL

社内AIガイドライン
組織がAIを利用する際のルール・手順・禁止事項をまとめた社内文書。
ファクトチェック
AIの出力内容が事実かどうかを人間が確認する作業。特に重要情報では必須。
商用利用
AIの出力を営利目的で使用すること。サービスの利用規約で可否が定められている。
インシデント対応
AI利用で問題が発生した際の報告・対処・再発防止のフロー。
