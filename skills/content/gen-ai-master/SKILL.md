# Gen AI Master — AI e-learning 教材自動生成エージェント

生成AI（Generative AI）に関するe-learning教材を、カリキュラム番号の入力だけで自動生成する。

## アーキテクチャ

```
gen-ai-master（メインエージェント / オーケストレーター）
    ├── Phase 0: カリキュラム取得（スプシ読み込み）
    ├── Phase 1: 直列
    │   ├── researcher     — Webリサーチ
    │   └── writer         — 教材執筆・校正
    ├── Phase 2: 並列（document.md完成後に同時実行）
    │   ├── pdf-maker      — 教材PDF化
    │   ├── glossary-maker — 用語集PDF化
    │   └── slide-maker    — /slide-creator で画像スライドPDF
    └── Phase 3: 直列
        └── gdrive-uploader — Googleドライブ保存
```

## 処理フロー

```
カリキュラム番号入力（例：1-1-1）
    ↓
━━ Phase 0: カリキュラム全体像の確認 ━━━━━━━━━━━━━━━
    スプレッドシートからCSVを取得し、最新のカリキュラム全体像を把握する。
    - この章の位置づけ（前の章・次の章との関係）
    - このセクション内の他のレッスンとの関連
    - レベル（基礎/実践/応用）の確認
    - トピックの概要説明を確認
    ↓
━━ Phase 1: 直列（土台づくり）━━━━━━━━━━━━━━━━━━
    ① リサーチ：トピックについてWeb検索で深く調査
        ↓
    ② 教材作成：調査結果を教材ドキュメント（Markdown）にまとめ、
       writing-guide.md に従ってやさしく校正
        → document.md として保存
    ↓
━━ Phase 2: 並列（成果物を同時生成）━━━━━━━━━━━━━━━
    ┌─ pdf-maker:      document.md → 教材PDF
    │   例：1-1.教材.pdf
    │
    ├─ glossary-maker:  教材から用語を抽出 → 用語集PDF
    │   例：用語集.pdf（セクション単位で共有）
    │
    └─ slide-maker:     /slide-creator を呼び出し
        document.md → 10ページの画像スライド → PDF
        例：1-1.スライド.pdf
    ↓
━━ Phase 3: 直列（アップロード）━━━━━━━━━━━━━━━━━
    チャプターフォルダごとGoogleドライブに保存
```

## 出力フォルダ構成

**命名ルール: 章番号は省略する。** ファイルはチャプターフォルダ内にあるため、章番号は冗長。
`{節-番号}` の2桁で統一する（節=セクション番号、番号=レッスン番号）。

```
outputs/
└── 第1章 AIの基礎知識/
    ├── 1.AIとは/
    │   ├── 1-1.教材.pdf
    │   ├── 1-1.スライド.pdf
    │   ├── 1-2.教材.pdf
    │   ├── 1-2.スライド.pdf
    │   ├── 1-3.教材.pdf
    │   ├── 1-3.スライド.pdf
    │   └── 用語集.pdf
    ├── 2.AIの仕組み/
    │   └── ...
    └── 3.深層学習(Deep Learning)/
        └── ...
```

### ファイル命名規則

章番号を省略し、`{節-番号}.{種類}` の短い形式で統一する。レッスン名はファイル名に含めない。

| 種類 | ファイル名パターン | 例（カリキュラム番号1-1-1の場合） |
|---|---|---|
| 教材PDF | `{節-番号}.教材.pdf` | `1-1.教材.pdf` |
| スライドPDF | `{節-番号}.スライド.pdf` | `1-1.スライド.pdf` |
| 用語集PDF | `用語集.pdf` | `用語集.pdf` |
| 教材MD | `{節-番号}_document.md` | `1-1_document.md` |

### フォルダ命名規則

章番号はチャプターフォルダに含まれるため、セクションフォルダでも省略する。

| レベル | フォルダ名パターン | 例 |
|---|---|---|
| チャプター | `第{N}章 {章タイトル}` | `第1章 AIの基礎知識` |
| セクション | `{節}.{セクションタイトル}` | `1.AIとは` |

## 各ステップの詳細

### Phase 0: カリキュラム全体像の確認

1. スプシからCSVを取得
2. 入力されたカリキュラム番号（例：1-1-1）の位置を特定
3. 以下を把握する：
   - **この章の全体像**: 何セクション、何レッスンあるか
   - **前後関係**: 前のレッスンで何を学んだか、次で何を学ぶか
   - **レベル**: 基礎/実践/応用
   - **概要説明**: スプシに記載されたトピックの概要
4. これらの情報を以降のPhaseに渡す（特にリサーチと教材作成で使う）

### Phase 1-①: リサーチ（researcher）

カリキュラム番号からトピック名を取得し、WebSearchで以下を調査する：
- トピックの定義・概要
- 具体例・事例
- 歴史的背景（該当する場合）
- 最新動向（2025-2026年時点）
- 初学者が誤解しやすいポイント

Phase 0で把握した前後のレッスンとの関係も踏まえ、
重複を避けつつこのレッスン固有の内容に焦点を当てる。

### Phase 1-②: 教材作成（writer）

リサーチ結果をもとに、教材ドキュメント（Markdown）を作成する。

**構成の参考:** `references/document-sample.md`

#### ボリューム基準（重要）

参照教材（`第1章 AIの基礎知識 1.AIとは（教材ドキュメント）.pdf`）と同等のボリュームを目指す：

- **1レッスンあたり PDF 5〜6ページ相当**（Markdown で 3,000〜5,000文字）
- **セクション数: 7〜9個**（① 〜 ⑦〜⑨ + まとめ）
- **各セクション内に太字の小見出し（### レベル）を 2〜3個** 入れる
- **具体例・事例を最低 10個以上** 散りばめる
- 薄い教材にしないこと。初学者が読んで「なるほど」と思える深さが必要

#### 必須構成要素

1. **ひとことで言うと** — 冒頭に一文で要約
2. **番号付きセクション ① ② ③ ... ⑦〜⑨** — 各セクション内の構成:
   - 太字の小見出し（### レベル）で 2〜3 のサブトピックに分割
   - 各サブトピック: 概要 → 具体例 → ポイント の流れ
   - セクション末に「ポイント：」で要点をまとめる（任意）
3. **「よくある誤解」セクション** — 必ず含める
   - 「〜→ 間違い」の形式で 3〜5個の誤解を列挙
   - 各誤解に正しい理解を簡潔に説明
4. **「最新動向」セクション** — トピックに応じて含める
   - 2025-2026年時点の具体的なサービス名・企業名を挙げる
5. **まとめ** — 学んだことの振り返り（3〜5文）

#### 校正ルール（`references/writing-guide.md` に従う）

- 専門用語は初出時に必ずかみ砕いた説明を添える
- 日常の例え話を積極的に使う（「〜で例えると」「〜に似ています」のフレーズで導入）
- 一文を短くする（50文字以内を目安）
- 「〜です。〜ます。」の丁寧語で統一
- 太字（**強調**）で重要ポイントを視覚的に目立たせる（各セクション 1〜2箇所）
- 読者を「あなた」と呼びかけない
- 比較表（| 表形式 |）を適宜使い、情報を整理する

完成したら `document.md` として保存する。

### Phase 2-A: 教材PDF化（pdf-maker）

`scripts/convert_to_pdf.py` を使って document.md → PDF 変換する。

**ファイル名:** `{節-番号}.教材.pdf`
例: `1-1.教材.pdf`

### Phase 2-B: 用語集生成（glossary-maker）

教材内の重要キーワードを抽出し、以下の形式でまとめる：

| キーワード | 一言解説 | 詳しい説明 |
|---|---|---|
| AI（人工知能） | 人間の知的作業をコンピューターで再現する技術の総称 | Artificial Intelligenceの略。1956年の... |

- 用語集はセクション単位（例：1-1）でまとめる
- 同じセクション内の他レッスンの教材がすでにある場合、それらの用語も統合する
- PDF化して `用語集.pdf` として保存

### Phase 2-C: 画像スライド生成（slide-maker）

`/slide-creator` スキルを呼び出して、document.md から10ページの画像スライドを生成する。

処理の流れ：
1. document.md を読み込み、10ページのスライド構成を設計
2. nanobanana API で各ページの画像を生成（フレームなし）
3. add_frame.py で統一フレームを追加
4. PDFに結合

**ファイル名:** `{節-番号}.スライド.pdf`
例: `1-1.スライド.pdf`

### Phase 3: Googleドライブ保存（gdrive-uploader）

`scripts/upload_to_gdrive.py` を使い、チャプターフォルダごとGoogleドライブに保存する。

保存先の構造（Googleドライブ上）：
```
Gen AI Master/
└── 第1章 AIの基礎知識/
    ├── 1.AIとは/
    │   ├── 1-1.教材.pdf
    │   ├── 1-1.スライド.pdf
    │   ├── 1-2.教材.pdf
    │   ├── 1-2.スライド.pdf
    │   ├── 1-3.教材.pdf
    │   ├── 1-3.スライド.pdf
    │   └── 用語集.pdf
    └── ...
```

## 注意事項

- **毎回スプシを確認**: カリキュラム全体像は毎回最新版を取得する
- **全体を見て個を作る**: 前後のレッスンとの重複を避け、このレッスン固有の内容に集中する
- **品質最優先**: 速度より正確さとわかりやすさを重視
- **用語集はセクション単位**: 同じセクション内のレッスン間で共有
- **並列実行**: Phase 2の3つのタスクは同時に実行して効率化
- **スプシが変わったら対応**: カリキュラム構成が変わっていたら、フォルダ構成もそれに合わせる
