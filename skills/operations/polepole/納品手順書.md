# POLEPOLE 入園手続き自動化システム - セットアップ手順書

## 概要

このシステムは、Google Apps Script（GAS）を使って以下を自動化します：

1. **Googleフォーム** - 入園手続き・アセスメント情報の入力（3分岐：未就学児/就学児/継続）
2. **スプレッドシート** - 回答データの自動振り分け（7シート）
3. **Googleドライブ** - 児童ごとのフォルダ自動作成
4. **契約書** - テンプレートからの自動生成・PDF変換
5. **メール** - 保護者への書類送付（下書き保存 or 即時送信）

## セットアップ手順

### STEP 1：スプレッドシートを作成

1. Googleドライブを開く
2. 「新規」→「Googleスプレッドシート」→「空白のスプレッドシート」
3. ファイル名を「POLEPOLE_入園データ管理」に変更

### STEP 2：GASプロジェクトを作成

1. スプレッドシートのメニューから「拡張機能」→「Apps Script」を開く
2. 左側のファイル一覧で、デフォルトの `コード.gs` を削除
3. 以下のファイルを1つずつ追加する：
   - 「＋」ボタン →「スクリプト」を選択
   - ファイル名を入力（拡張子 `.gs` は不要）
   - 対応するコードを貼り付け

| 追加するファイル名 | 対応ファイル |
|---|---|
| `00_config` | `gas/00_config.gs` |
| `01_setup` | `gas/01_setup.gs` |
| `02_formBuilder` | `gas/02_formBuilder.gs` |
| `03_triggerHandler` | `gas/03_triggerHandler.gs` |
| `04_sheetManager` | `gas/04_sheetManager.gs` |
| `05_driveManager` | `gas/05_driveManager.gs` |
| `06_contractManager` | `gas/06_contractManager.gs` |
| `07_emailManager` | `gas/07_emailManager.gs` |

### STEP 3：初期セットアップを実行

1. GASエディタで、上部の関数選択プルダウンから `initialSetup` を選択
2. 「実行」ボタンをクリック
3. 権限の承認ダイアログが表示されるので「許可」をクリック
   - 「このアプリは確認されていません」と出たら「詳細」→「〇〇（安全でないページ）に移動」
4. セットアップ完了のダイアログが表示されます

### STEP 4：動作確認

1. スプレッドシートを再読み込み（F5キー）
2. メニューバーに「POLEPOLE管理」が表示されることを確認
3. 「POLEPOLE管理」→「フォームURLを表示」でフォームURLを確認
4. テスト回答を送信し、各シートにデータが振り分けられることを確認

## 日常の操作方法

### フォームの共有

1. スプレッドシートの「POLEPOLE管理」メニュー →「フォームURLを表示」
2. 「回答URL」を保護者に共有（メール、LINE等）

### 契約書の作成

1. スプレッドシートの「POLEPOLE管理」メニュー →「契約書を作成」
2. 児童を選択して「契約書を作成」ボタンをクリック
3. Googleドキュメントが開くので、署名部分を調整
4. 完了後、メール送信へ

### メールの送信

1. スプレッドシートの「POLEPOLE管理」メニュー →「メール下書きを作成」
2. 児童を選択、メールの種類を選択
3. 「下書き保存」→ Gmailで確認してから送信
4. または「今すぐ送信」で直接送信

## 設定の変更方法

### 施設名や各種名称を変更する場合

`00_config.gs` の `CONFIG` オブジェクト内の値を変更してください。

```javascript
const CONFIG = {
  FACILITY_NAME: 'POLEPOLE',  // ← ここを変更
  FORM_TITLE: 'POLEPOLE 入園手続き・アセスメントフォーム',  // ← ここを変更
  ...
};
```

### フォームの質問を変更する場合

`02_formBuilder.gs` 内の各セクション構築関数を編集してください：
- `buildBasicInfoSection()` - 基本情報
- `buildPreschoolSection()` - 未就学児アセスメント
- `buildSchoolAgeSection()` - 就学児アセスメント
- `buildContinuationSection()` - 継続アセスメント
- `buildCommonEndSection()` - 写真・送迎（共通）

**注意**: フォームの質問を変更した場合は、`03_triggerHandler.gs` のデータ振り分け処理も合わせて更新してください。

### スプレッドシートのシート構成を変更する場合

`04_sheetManager.gs` の `setupSpreadsheet()` 関数を編集してください。

## トラブルシューティング

| 症状 | 対処法 |
|------|--------|
| メニューが表示されない | スプレッドシートを再読み込み（F5） |
| フォーム送信してもシートに反映されない | GASエディタで `onFormSubmit` のトリガーが設定されているか確認 |
| 権限エラーが出る | GASエディタで再度実行し、権限を再承認 |
| メールが送信できない | Gmail APIの利用上限（1日500通）を確認 |

## Google Workspace有料プランへの移行後

有料プランに移行すると以下が利用可能になります：

- **Google Workspace Essentials Starter以上**:
  - カスタムドメインのメールアドレスで送信可能
  - Google Vaultによるデータ保全

- **電子署名について**:
  - Google Workspace有料プランではサードパーティの電子署名サービス（DocuSign等）との連携が可能
  - 現在のシステムでは契約書をPDFで送付 → 印刷・署名・返送 の流れ
  - 有料プラン移行後、電子署名機能を追加実装可能
