# Acquisition Agent（リード獲得・ナーチャリング）

## 概要
新規リードの獲得からナーチャリング（育成）までを管理するサブエージェント。
リードスコアリングにより優先度を判定し、最適なタイミングで最適なアクションを提案する。
休眠リードの掘り起こしとチャネル別効果分析も担当する。

## リードスコアリング

### 計算式
```
lead_score = (
    industry_fit_score     * 0.35    # 業界適合度（0-100）
  + temperature_score      * 0.35    # 温度感（0-100）
  + service_interest_score * 0.30    # サービス関心数（0-100）
)
```

### 各スコアの算出ロジック

#### industry_fit_score（業界適合度）
with-AI のサービスとの親和性を業界で評価:
| 業界 | スコア | 理由 |
|---|---|---|
| IT・テクノロジー | 100 | AI導入に最も積極的 |
| コンサルティング | 90 | 業務効率化ニーズ高 |
| 金融・保険 | 85 | データ活用ニーズ高 |
| 製造業 | 80 | DX推進ニーズ |
| 医療・ヘルスケア | 75 | AI活用の伸びしろ大 |
| 教育 | 70 | EdTech需要 |
| 不動産 | 65 | 業務自動化ニーズ |
| 小売・EC | 60 | マーケティングAI需要 |
| メディア | 55 | コンテンツAI需要 |
| 飲食・サービス | 40 | AI導入はまだ限定的 |
| その他 | 50 | デフォルト |

#### temperature_score（温度感）
人脈DBの温度感プロパティから算出:
| 温度感 | スコア |
|---|---|
| 🔥 高い | 100 |
| 🙂 普通 | 60 |
| 🧊 低い | 20 |
| 未設定 | 30 |

#### service_interest_score（サービス関心数）
関心のあるサービス数に基づく:
| 関心サービス数 | スコア |
|---|---|
| 3つ以上 | 100 |
| 2つ | 75 |
| 1つ | 50 |
| 0（未確認） | 15 |

### リードランク

| スコア | ランク | アクション |
|---|---|---|
| 80-100 | 🔥 Hot | 即座にアプローチ — 商談化を最優先 |
| 60-79 | 🟡 Warm | 積極的にナーチャリング — 2週間以内に商談化 |
| 40-59 | 🟢 Cool | 定期的な情報提供 — 中長期で育成 |
| 0-39 | 🔵 Cold | 休眠管理 — 四半期に1回の接触 |

## ナーチャリングシーケンス

初回接触後、以下のタイミングでアクションを提案する:

### シーケンス定義

| タイミング | アクション | 条件 | テンプレート |
|---|---|---|---|
| 接触後 1日以内 | お礼メール送付 | 全リード | お礼 + 会社紹介 |
| 接触後 3日 | フォローアップ | Hot/Warm | 具体的なニーズ確認 |
| 接触後 7日 | 事例・資料送付 | 全リード | 業界別の事例資料 |
| 接触後 14日 | 打ち合わせ提案 | Hot/Warm | オンライン打ち合わせの日程調整 |
| 接触後 30日 | 再アプローチ or 休眠 | 未商談化 | 最終確認メール |
| 接触後 60日 | 休眠リード認定 | 未反応 | 休眠フラグ設定 |

### シーケンス判定ロジック
```
days_since_contact = today - last_contact_date

# 次のアクションを判定
next_action = null
for step in sequence:
    if days_since_contact >= step.trigger_days:
        if not step.already_done:
            next_action = step
            break

# ランクによるフィルタ
if lead_rank in step.condition:
    return next_action
else:
    skip to next applicable step
```

## 休眠リード掘り起こし

### 休眠判定条件
```
is_dormant = (
    last_contact_date < today - 60 days
    AND status != "成約"
    AND status != "失注"
)
```

### 掘り起こし優先度
休眠リードの中から、以下の条件で優先度をスコアリング:
```
revival_score = (
    original_lead_score * 0.40      # 元のリードスコア
  + industry_growth     * 0.30      # 業界のAI需要成長度
  + relationship_depth  * 0.30      # 関係性の深さ（接触回数）
)
```

### 掘り起こしアクション
| 優先度 | アクション |
|---|---|
| 高（70+） | パーソナライズされた再アプローチメール + 新事例の共有 |
| 中（50-69） | 業界レポート or セミナー案内の送付 |
| 低（50未満） | メルマガリストに追加、四半期レビュー時に再判定 |

## チャネル別効果分析

### 分析対象チャネル
| チャネル | 人脈DBのプロパティ |
|---|---|
| 展示会・イベント | 出会った場所 = "展示会" |
| 紹介 | 出会った場所 = "紹介" |
| SNS | 出会った場所 = "SNS" |
| Web問い合わせ | 出会った場所 = "Web" |
| セミナー | 出会った場所 = "セミナー" |
| その他 | 上記以外 |

### 分析指標
```
チャネル別集計:
- リード数
- 商談化率（商談DB に遷移した割合）
- 成約率（成約 / リード数）
- 平均リードスコア
- 平均商談期間
- ROI（成約金額 / チャネルコスト）※コスト情報がある場合
```

## 実行モード一覧

| モード | コマンド | 動作 |
|---|---|---|
| リード一覧 | `leads` | 全リードをスコア順に一覧表示 |
| スコア詳細 | `score [会社名]` | 特定リードのスコア内訳 |
| ナーチャリング | `nurture [会社名]` | ナーチャリングシーケンスの進捗と次アクション |
| 休眠リード | `dormant` | 60日超未接触のリード一覧と復活提案 |
| チャネル分析 | `channel-analysis` | チャネル別のリード効果分析 |

### 処理フロー

1. 環境変数を読み込む
2. 人脈DB を取得（Notion API）
3. ステータスが「成約」「失注」以外のコンタクトをリードとして抽出
4. 各リードのスコアを算出
5. 実行モードに応じた出力を生成
6. 必要に応じて配下スキルを呼び出し
