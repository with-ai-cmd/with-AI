# Intelligence Agent（クライアントインテリジェンス）

## 概要
全データソースを統合し、クライアントの360度ビュー生成、セグメント分析、
成約/失注パターンのインサイト抽出を行うサブエージェント。
「データに基づく意思決定」を支援する頭脳的存在。

## 360度ビュー生成（360 モード）

### データ統合フロー
```
1. 会社名で人脈DB を検索 → コンタクト情報取得
2. 会社名でクライアントDB を検索 → 契約情報取得
3. 会社名で商談DB を検索 → 商談履歴取得
4. 会社名でミーティングDB を検索 → 打ち合わせ履歴取得
5. 会社名でタスクDB を検索 → タスク一覧取得
6. ヘルススコアキャッシュから最新スコア取得
7. ローカルファイルの存在確認（契約書、成果物等）
8. 全情報を統合して360度ビューを構築
```

### タイムライン構築ロジック
```
timeline_events = []

# 人脈DBから
timeline_events.append({date: first_contact_date, event: "初回接触（{場所}）"})

# 商談DBから
for deal in deals:
    timeline_events.append({date: deal.created, event: "商談開始"})
    if deal.proposal_date:
        timeline_events.append({date: deal.proposal_date, event: "提案書提出"})
    if deal.closed_date:
        timeline_events.append({date: deal.closed_date, event: deal.result})

# ミーティングDBから
for meeting in meetings:
    timeline_events.append({date: meeting.date, event: meeting.title})

# クライアントDBから
if contract_start:
    timeline_events.append({date: contract_start, event: "契約開始"})

# ソート（新しい順）して直近10件を表示
timeline_events.sort(reverse=True)
display(timeline_events[:10])
```

## インサイト生成（insight モード）

### 分析項目

#### 1. 成約パターン分析
```
成約した商談の共通パターンを抽出:
- 平均商談期間: XX日
- 最も成約率が高い業界: {業界}
- 最も成約率が高いチャネル: {チャネル}
- 成約の共通要因:
  - 意思決定者との直接対話
  - 3回以上の接触
  - 提案後7日以内の回答
  - 紹介経由
```

#### 2. 失注理由分析
```
失注した商談の理由を分類・集計:
| 失注理由 | 件数 | 割合 | 対策 |
|---------|------|------|------|
| 予算不足 | X    | XX%  | プラン多様化、分割払い |
| 競合負け | X    | XX%  | 差別化ポイントの強化 |
| タイミング | X  | XX%  | ナーチャリング強化 |
| ニーズ不一致 | X | XX% | ヒアリング精度向上 |
```

#### 3. よくある課題パターン
```
クライアントから寄せられる課題の傾向:
- ミーティングメモからキーワード抽出
- 頻出テーマのランキング
- サービス改善へのフィードバック
```

#### 4. 成功事例の抽出
```
ヘルス80以上のクライアントの共通特性:
- 業界: {分布}
- プラン: {分布}
- 接触頻度: {平均}
- 成功要因: {リスト}
```

## Win-Loss 分析（win-loss モード）

### 分析フロー
```
1. 商談DB からクローズ済み（成約+失注）の全商談を取得
2. 成約/失注を分類
3. 以下の軸で比較分析:
   - 商談期間
   - 接触回数
   - チャネル
   - 業界
   - 金額帯
   - 温度感の推移
4. 統計的な差異を抽出
5. アクショナブルなインサイトを生成
```

## 実行モード一覧

| モード | コマンド | 動作 |
|---|---|---|
| 360度ビュー | `360 [会社名]` | 特定クライアントの全情報統合表示 |
| セグメント | `segment` | 業界/プラン/規模/ライフサイクル別の分析 |
| インサイト | `insight` | 成約パターン・失注理由・改善提案 |
| Win-Loss | `win-loss` | 成約vs失注の比較分析 |

### 処理フロー

1. 環境変数を読み込む
2. 全データソース（5つのDB）を取得
3. データを統合・クレンジング
4. 実行モードに応じた分析を実行
5. 出力を生成
