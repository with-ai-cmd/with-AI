# Relationship Agent（リレーション管理）

## 概要
全クライアント・コンタクトとの関係性を維持・強化するサブエージェント。
「誰に・いつ・どのように接触すべきか」を管理し、
フォローアップの漏れを防止し、紹介ネットワークを可視化する。

## タッチポイント管理

### 接触頻度しきい値

対象カテゴリ別に最適な接触頻度を定義:

| カテゴリ | 理想頻度 | 注意アラート | 警告アラート |
|---|---|---|---|
| 契約中クライアント | 2週間以内 | 21日超 | 30日超 |
| 商談中コンタクト | 1週間以内 | 10日超 | 14日超 |
| ナーチャリング中 | 2週間以内 | 21日超 | 30日超 |
| 休眠前コンタクト | 1ヶ月以内 | 45日超 | 60日超 |
| 紹介者・パートナー | 1ヶ月以内 | 45日超 | 60日超 |

### 最終接触日の特定ロジック
```
last_contact_date = MAX(
    contact_db.last_contact_date,           # 人脈DBの最終接触日
    MAX(meeting_db.date WHERE company = X), # ミーティングDBの最新日付
    MAX(task_db.completed_date WHERE company = X AND type = "連絡")
)
```

### タッチポイント一覧表示
```
days_since_contact = today - last_contact_date

status = case:
    days_since_contact <= ideal_frequency:      "🟢 良好"
    days_since_contact <= warning_threshold:     "🟡 注意"
    days_since_contact > warning_threshold:      "🔴 要対応"
```

## フォローアップ自動化

### 未接触アラート

以下の条件でアラートを生成:

| 対象 | 条件 | アラートレベル |
|---|---|---|
| クライアント | 30日超未接触 | 🔴 警告 — 即時フォロー |
| クライアント | 21日超未接触 | 🟡 注意 — 今週中にフォロー |
| コンタクト（商談中） | 14日超未接触 | 🔴 警告 — 商談が冷める |
| コンタクト（商談中） | 10日超未接触 | 🟡 注意 — フォロー推奨 |
| コンタクト（その他） | 14日超未接触 | 🟡 注意 — 関係維持のフォロー |

### フォローアップ優先度
```
follow_priority = (
    urgency_score      * 0.40    # 未接触日数が長いほど高い
  + importance_score   * 0.35    # クライアント>商談中>ナーチャリング
  + recoverability     * 0.25    # リカバリーの可能性（温度感）
)
```

#### urgency_score
| 超過日数 | スコア |
|---|---|
| しきい値の2倍超 | 100 |
| しきい値の1.5倍超 | 80 |
| しきい値超 | 60 |
| しきい値以内 | 0（アラート対象外） |

#### importance_score
| カテゴリ | スコア |
|---|---|
| 契約中クライアント | 100 |
| 商談中コンタクト | 80 |
| ナーチャリング中 | 50 |
| その他コンタクト | 30 |

### メール生成ロジック
```
1. 会社名からコンタクト情報を取得（人脈DB）
2. 直近のミーティング情報を取得（ミーティングDB）
3. 未完了タスクを取得（タスクDB）
4. テンプレートタイプを判定:
   - ミーティング直後 → お礼
   - 未接触期間が長い → フォローアップ
   - 商談中で提案段階 → 提案
   - 契約記念日付近 → 記念
5. テンプレートに情報を注入して下書きを生成
```

## 紹介ネットワークマッピング

### 紹介関係の追跡
人脈DBの「紹介者」プロパティを基にネットワークを構築:

```
紹介ネットワーク:
  田中さん（AAA社）
    ├── 佐藤さん（BBB社）→ 成約
    └── 鈴木さん（CCC社）→ 商談中

  山田さん（DDD社）
    └── 高橋さん（EEE社）→ 初回接触
```

### 紹介ハブの特定
```
referral_hub_score = (
    referral_count         * 0.40    # 紹介件数
  + referral_conversion    * 0.35    # 紹介先の成約率
  + relationship_health    * 0.25    # 紹介者自身のヘルススコア
)
```

### 紹介依頼タイミング
以下の条件を全て満たすクライアントに紹介依頼を提案:
```
can_request_referral = (
    health_score >= 80
    AND contract_duration >= 90 days
    AND last_referral_request > 90 days ago (or never)
    AND recent_success_story exists
)
```

## 実行モード一覧

| モード | コマンド | 動作 |
|---|---|---|
| フォロー一覧 | `follow` | フォローアップが必要な全一覧 |
| タッチポイント | `touchpoint [会社名]` | 特定クライアント/コンタクトの接触履歴 |
| メール下書き | `draft-email [会社名]` | フォローメールの自動生成 |
| ネットワーク | `network` | 紹介ネットワークの可視化 |

### 処理フロー

1. 環境変数を読み込む
2. 人脈DB・クライアントDB を取得
3. ミーティングDB から接触履歴を取得
4. 各コンタクト/クライアントの最終接触日を算出
5. しきい値と比較してアラートを生成
6. 実行モードに応じた出力を生成
