# Pipeline Agent（パイプライン管理）

## 概要
商談パイプライン全体を管理するサブエージェント。
ファネルの可視化、各フェーズの滞留検知、確度スコアリング、受注予測を通じて
「いつ・どの商談に・どう介入すべきか」を明確にする。

## ファネル定義

商談は以下の4フェーズ + 2終了状態で管理する:

```
初回接触 → 商談中 → 提案済み → 成約（Won）
                              → 失注（Lost）
```

### フェーズ判定基準

| フェーズ | 条件 | 次フェーズへの遷移条件 |
|---|---|---|
| 初回接触 | コンタクト登録済み、商談意思あり | 具体的なニーズヒアリング完了 |
| 商談中 | ニーズ確認済み、要件整理中 | 提案書提出 or 見積もり提示 |
| 提案済み | 提案書/見積もり提出済み | 先方から回答（承諾 or 辞退） |
| 成約 | 契約締結 | Contract Agent へ引き継ぎ |
| 失注 | 辞退・競合負け・予算なし等 | 失注理由記録 → Intelligence Agent へ |

## 確度スコアリング

### 計算式
```
probability_score = (
    temperature_score    * 0.35    # 温度感（0-100）
  + frequency_score      * 0.25    # 接触頻度（0-100）
  + recency_score        * 0.20    # 最終接触からの経過日数（0-100）
  + budget_score         * 0.10    # 予算感の明確さ（0-100）
  + decision_score       * 0.10    # 意思決定者の関与度（0-100）
)
```

### 各スコアの算出ロジック

#### temperature_score（温度感）
商談DBの温度感プロパティから算出:
| 温度感 | スコア |
|---|---|
| 🔥 高い | 100 |
| 🙂 普通 | 60 |
| 🧊 低い | 20 |
| 未設定 | 40 |

#### frequency_score（接触頻度）
直近30日間のミーティング・連絡回数:
| 回数 | スコア |
|---|---|
| 4回以上 | 100 |
| 3回 | 80 |
| 2回 | 60 |
| 1回 | 40 |
| 0回 | 10 |

#### recency_score（最終接触の新しさ）
最終接触日からの経過日数:
| 経過日数 | スコア |
|---|---|
| 0-3日 | 100 |
| 4-7日 | 80 |
| 8-14日 | 60 |
| 15-21日 | 40 |
| 22-30日 | 20 |
| 31日以上 | 5 |

#### budget_score（予算感の明確さ）
| 状態 | スコア |
|---|---|
| 金額が具体的に議論済み | 100 |
| 予算枠あり（範囲提示） | 70 |
| 予算検討中 | 40 |
| 予算未確認 | 10 |

#### decision_score（意思決定者の関与度）
| 状態 | スコア |
|---|---|
| 意思決定者と直接やり取り | 100 |
| 意思決定者を紹介済み | 70 |
| 担当者レベルのみ | 40 |
| 不明 | 20 |

### 確度ランク

| スコア | ランク | 意味 |
|---|---|---|
| 80-100 | A | 高確度 — 受注見込み大 |
| 60-79 | B | 中確度 — 積極フォロー |
| 40-59 | C | 低確度 — 要テコ入れ |
| 0-39 | D | 極低 — 見極め or 撤退検討 |

## 滞留検知ロジック

各フェーズに滞留日数のしきい値を設定し、超過した商談をアラートする。

| フェーズ | 注意（黄色） | 警告（赤色） |
|---|---|---|
| 初回接触 | 7日超 | 14日超 |
| 商談中 | 14日超 | 30日超 |
| 提案済み | 7日超 | 14日超 |

### 滞留判定
```
stale_days = today - phase_entered_date

if stale_days > red_threshold:
    alert = "🔴 警告: {会社名} が{フェーズ}で{stale_days}日滞留中"
    action = "即座にアクションを取るか、フェーズを見直してください"
elif stale_days > yellow_threshold:
    alert = "🟡 注意: {会社名} が{フェーズ}で{stale_days}日経過"
    action = "フォローアップを検討してください"
```

## 受注予測計算

### 月次受注予測
```
weighted_pipeline = Σ (deal_amount × probability_score / 100)
```

## 実行モード一覧

| モード | コマンド | 動作 |
|---|---|---|
| ファネル表示 | `status` | パイプライン全体のファネルビュー + 商談一覧 |
| 受注予測 | `forecast` | 月次の加重パイプライン受注予測 |
| 滞留検知 | `stale` | しきい値超過の商談アラート |
| 商談詳細 | `detail [会社名]` | 特定商談の詳細情報・タイムライン |

### 処理フロー

1. 環境変数を読み込む
2. 商談DB を全件取得（Notion API）
3. 各商談について:
   a. フェーズを判定
   b. 確度スコアを算出（ミーティングDB も参照）
   c. 滞留日数を計算
4. 実行モードに応じた出力を生成
5. `{{WITHAI_ROOT}}/cco/state/pipeline.json` を更新
