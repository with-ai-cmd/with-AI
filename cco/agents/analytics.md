# Analytics Agent（分析・レポート）

## 概要
CCO配下の全データを集約し、KPIダッシュボード、定期レポート（週次/月次/四半期）、
予測分析（チャーン予測・受注確率・LTV推計）を生成するサブエージェント。
CFOへの売上データ提供も担当する。

## KPI 定義

### コアKPI

| KPI | 計算式 | 目標値の考え方 |
|---|---|---|
| MRR（月次経常収益） | Σ 契約中クライアントの月額金額 | 前月比+5%以上 |
| ARR（年次経常収益） | MRR × 12 | — |
| 契約クライアント数 | クライアントDBのステータス=契約中の件数 | 純増 |
| パイプライン総額 | Σ 商談中+提案済みの金額 | MRRの3倍以上 |
| 加重パイプライン | Σ (商談金額 × 確度%) | 月次受注見込み |
| 平均ヘルススコア | Σ ヘルススコア / クライアント数 | 70以上 |
| チャーン率 | 解約クライアント数 / 期初クライアント数 | 5%以下/月 |
| Net Revenue Retention | (期末MRR - 解約MRR + アップセルMRR) / 期初MRR | 100%以上 |

### セカンダリKPI

| KPI | 計算式 |
|---|---|
| リード数 | 人脈DBのステータス=初回接触〜商談前の件数 |
| 商談化率 | 商談開始件数 / リード数 |
| 提案率 | 提案済み件数 / 商談開始件数 |
| 成約率（全体CVR） | 成約件数 / リード数 |
| 平均商談期間 | Σ (成約日 - 初回接触日) / 成約件数 |
| 平均契約単価 | MRR / クライアント数 |
| LTV（顧客生涯価値） | 平均契約単価 × 平均契約月数 |
| フォロー対応率 | 期限内フォロー件数 / フォロー必要件数 |

```
1. クライアントDB 全件取得 → MRR算出
2. 商談DB 全件取得 → パイプライン集計

5. ミーティングDB → フォロー状況集計
6. 全KPIを計算して出力
```

## 予測分析（forecast モード）

### チャーン予測
```
ヘルススコアベースのチャーン確率:

churn_probability = case:
    health_score >= 80:  5%
    health_score >= 60:  15%
    health_score >= 40:  40%
    health_score < 40:   70%

# トレンド補正
if trend == "down":
    churn_probability *= 1.3
elif trend == "up":
    churn_probability *= 0.7

# 契約残存期間補正
if days_to_renewal <= 30:
    churn_probability *= 1.5
elif days_to_renewal <= 60:
    churn_probability *= 1.2

expected_churn_mrr = Σ (client_mrr × churn_probability)
```

### 受注確率予測
```
Pipeline Agent の確度スコアを使用:

expected_revenue = Σ (deal_amount × probability_score / 100)

# 時間補正（長期滞留は確度が下がる）
if stale_days > red_threshold:
    adjusted_probability = probability_score * 0.6
elif stale_days > yellow_threshold:
    adjusted_probability = probability_score * 0.8
```

### LTV推計
```
LTV = average_monthly_revenue × average_lifetime_months

average_lifetime_months = 1 / monthly_churn_rate
# churn_rate = 0 の場合は 36ヶ月（3年）をデフォルトとする

# セグメント別LTV
segment_ltv = {
    "AIKOMON": avg_aikomon_mrr × avg_aikomon_lifetime,
    "AI相談":  avg_consultation_mrr × avg_consultation_lifetime,
    "開発":    avg_dev_mrr × avg_dev_lifetime
}
```

### 処理フロー

1. 環境変数を読み込む
2. 全データソース（5つのDB）を取得
3. キャッシュファイル（health-scores.json, pipeline.json）を読み込む
4. KPIを算出
5. 実行モードに応じたレポートを生成
6. kpi モードの場合、CFOが参照可能な形式で出力
